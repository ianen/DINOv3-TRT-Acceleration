# ADR-010 · V1.2 — ONNX Q/DQ Stripping for True Mixed-Precision

> 状态：**Implemented · Negative result**（2026-05-01 第二十四/二十五轮联合实施 + 实测）
> 触发：M1-progress 第十八轮（2026-05-01）— trtexec `--layerPrecisions` 在 explicit Q/DQ ONNX 上对 cosine 是 no-op，因为 Q/DQ 节点是 TRT 真正的精度边界。
> 依据：M1-progress 第十四轮（SmoothQuant + skip 16-19 ModelOpt disable，speed −30% 而 cos 仅 +0.003） + 第十八轮（trtexec layer_precisions:fp32 完全 no-op）联合证据。
> 期望：V1.2 实施后给项目提供"BF16 prefer ↔ INT8 SmoothQuant"之间的中间候选，潜在地把 cos_min 从 0.968 推到 0.99 阈值之上。

## 1. 决策点

如何在保留 SmoothQuant α=0.8 INT8 加速的前提下，让 layer 16-19（最敏感的最后 4 个 transformer block）真正以 BF16/FP16 执行，跨过 G2 cos_min ≥ 0.99 阈值？

## 2. 备选方案对比

| 方案 | V1.1 已验证 | 结论 |
|---|---|---|
| **A** ModelOpt `disable_quantizer` skip 16-19 | 第十四轮 | **Negative-ish**：cos +0.003 / speed −30%。SmoothQuant smoothing scale 已写入所有 Linear weight，disable 子集 quantizer 让那些层 implicit fallback 到 dequantize→fp32→quantize 往返，速度大跌 |
| **B** trtexec `--layerPrecisions=l16-19:fp32` 在 SmoothQuant Q/DQ ONNX | 第十八轮 | **No-op**：cos 与 full SmoothQuant 完全相同，speed 也几乎相同。Q/DQ 节点是真精度边界，per-layer override 只控制 kernel 内部计算精度但 boundary tensor 仍 INT8 |
| **C** trtexec `--int8 --bf16 --layerPrecisions=l16-19:bf16` | 第十八轮 | **Build 失败**：TRT 10.13 Myelin Fill 算子不支持 BF16 类型（仅 Int32/Float/Half/Int64），RoPE Constant 触发 assertion failure |
| **D** ONNX Q/DQ stripping（本 ADR） | **未实施** | 在 ONNX 层删除 layer 16-19 内部所有 Q/DQ 节点，让 TRT 自然以 fp 精度执行那些层 — V1.1 三条路径联合指向的唯一可能入口 |

## 3. 选择

**方案 D**：onnx-graphsurgeon 在 ONNX 层做 Q/DQ stripping。

## 4. 实施约束（V1.1 实验观察）

### 4.1 类型支持矩阵（TRT 10.13 + Blackwell sm_120）

| 精度 | Myelin Fill 支持 | DINOv3 NaN 风险 | 适用性 |
|---|:---:|:---:|---|
| FP32 | ✅ | 无 | safe baseline，但失去加速意义 |
| FP16 | ✅ | **NaN（已确认负例）** | 不可用 |
| BF16 | ❌ | 无 | Myelin Fill 阻塞，与 RoPE Constant 不兼容 |
| INT8 | ✅ | Q/DQ 边界塌缩 | SmoothQuant 主路径已用 |

**结论**：layer 16-19 内部精度选择面非常窄。FP32 是唯一既被 Myelin 支持又无 NaN 风险的"非 INT8"候选。

### 4.2 ONNX 节点结构（已 inspect）

SmoothQuant α=0.8 ONNX 共 4401 节点。layer 16-19（4 个 transformer block）共 808 节点，op-type 直方图（M1-progress 第十七轮）：

- **量化节点**：QuantizeLinear 48 + DequantizeLinear 48 = 96 个 Q/DQ 节点
- **核心计算**：MatMul 32 + Add 56 + LayerNormalization 8 + Softmax 4 = 100 个
- **辅助节点**：Constant 272 + Mul 60 + Unsqueeze 64 + Transpose 44 + Slice 32 + Cast 24 + Shape 20 + Gather 20 + Reshape 16 + Div 12 + Concat 32 + Neg 8 + Sub 4 + Erf 4 = 612 个

每 transformer block 内部 ~12 个 Q/DQ pairs（attention q/k/v/o + mlp fc1/fc2 各 1 input quantizer + 1 weight quantizer），4 blocks × 12 = 48 个 Q/DQ pairs ≈ 96 节点。

### 4.3 边界保留策略

> 📍 **2026-05-01 第二十四轮 ONNX inspect 实证修订**：原假设的"layer 15→16 / layer 19→20 边界 Q/DQ pair 需保留"经 `inspect_qdq_pairs_for_blocks.py` 在真实 SmoothQuant α=0.8 ONNX 上扫描验证不成立。

**实测结果**（`Code/Artifacts/reports/qdq_pairs_inspect_blocks_16-19.json`）：

```
qdq_counts: { internal: 48, boundary_input: 0, boundary_output: 0, total: 48 }
```

**结构观察**：所有 48 个 pairs 都是 layer 内部的 `input_quantizer` 或 `weight_quantizer`（每 Linear 各一对），按 6 Linear ops × 2 quantizers × 4 blocks = **48 pairs / 96 节点**精准匹配；ModelOpt SmoothQuant 导出**不产生 cross-block 边界 Q/DQ**，cross-block tensor 是 fp32（block 15 output 直接以 fp32 流入 block 16 的 LayerNorm，没有 Q→DQ 节点在 block 边界上）。

**简化的 V1.2 strip 策略**：

- 直接删除所有 48 个 internal Q→DQ pairs（96 节点）。
- 每个 pair 的 rewire：把 DequantizeLinear 的 output tensor 的下游 consumer 重连到 QuantizeLinear 的 input tensor。
- 无需考虑 boundary 保留，cross-block 路径在 ONNX 层已经是 fp32。
- ✅ 简化设计：strip 操作变成纯粹的 "delete + rewire" 而不是 "preserve boundary + delete internal"。

### 4.4 操作语义

- 删除 internal Q/DQ pair 后，那些边的 tensor 类型从"INT8 with scale"变成"fp32"。
- TRT 在 build 时遇到无 Q/DQ 包围的 MatMul/Add/LayerNorm 会用 fp 精度（fp32 或 fp16/bf16，取决于 builder flags）。
- 由于 BF16/FP16 都有问题（4.1 表），唯一选择是 `--int8 --noTF32`，让 layer 16-19 内部以 **FP32** 执行。

### 4.5 预期结果

| 指标 | full SmoothQuant α=0.8（V1.1 baseline） | 方案 D（V1.2 预测） |
|---|---:|---:|
| feat_layer_20 cos_mean | 0.982 | **0.99+ 期望** |
| feat_layer_20 cos_min | 0.968 | **0.985+ 期望** |
| trtexec b8 speedup vs FP32 | 3.48× | **2.0-2.8× 期望** |

speed 必然下降（layer 16-19 从 INT8 → FP32 increases compute），但精度有望跨过 G2 cos_min ≥ 0.99 阈值。如果实测后 speed 仍 ≥ 2.2× 且 cos_min ≥ 0.99，**这就是首个完整满足 G2 ideal region 的 INT8 路径候选**。

### 5.3 V1.2 结论 — Definitive Negative

V1.2 ONNX-level Q/DQ stripping 与 V1.1 ModelOpt PyTorch 层 disable_quantizer **产生几乎完全相同的精度/速度结果**（cos_min 差 0.0005，speed 差 0.02×）：

- ✅ **速度跨过 G2 阈值**：2.39× ≥ 2.2×
- ❌ **精度仍未跨过 G2 阈值**：cos_min 0.9705 < 0.99（缺口 0.0195）

**机制**：layer 16-19 内部 fp 计算无法 recover 来自 blocks 0-15 累积的 INT8 量化噪声。**真正的瓶颈不是"末段 4 个 block 的精度"，而是"前段 20 个 block 的 INT8 量化已污染了输入"**。两条 mixed-precision 路径在 ONNX 与 PyTorch 层做相同的事，因此结果等价。

如果未来要真正跨过 G2，必须从前段开始减少量化（例如 SmoothQuant 全模型 α 调整 + 训练时 QAT），不是末段 mixed-precision。这超出 V1.0.0 + V1.1 + V1.2 的工程范围，是 V1.3 / 论文阶段的研究问题。

## 6. 风险与已知未知

| 风险 | 缓解 |
|---|---|
| **TRT FP32 fallback 仍可能 fuse 进 INT8 内核**：Myelin pattern matcher 看到无 Q/DQ 包围的 MatMul 仍可能选 INT8 tactic | 用 `--precisionConstraints=obey --layerPrecisions=l16-19:fp32` 强制；或用 `setPrecision()` API |
| **onnx-graphsurgeon 接合错误**：删除 Q/DQ 后 in-edges 直接接到原 Q 的 destination（DequantizeLinear），需要正确 rewire 到 Q 的 input source | 测试覆盖 mini synthetic ONNX；远端 inspect 重生 ONNX 验证 isomorphism |
| **SmoothQuant smoothing scale 残留**：weight 已经 smooth 过，layer 16-19 weight 仍是 smoothed 版本（数值范围被压缩） | 实测：smoothing 是 weight-only transform，与 Q/DQ 解耦；不影响 fp32 kernel 数值正确性 |
| **r336/r518 多分辨率扩展成本**：方案 D 在 r224 验证后可能需要 r336/r518 重做 | 第一阶段只做 r224；如果 r224 negative，不再扩展；如果 positive，再考虑 r336/r518 |

## 7. 实施判定门槛

V1.2 方案 D 应在以下条件满足时实施：

- 有 0.5-1 工程日预算专门用于 onnx-graphsurgeon 工程。
- ImageNet val 数据源已 unblock（让 cosine eval 跨过 Imagenette 局限性）。
- 或：作为 V1.1 stretch 之外的独立技术备忘任务，不在主交付时间线压力下。

否则保留为 future work，引用本 ADR 作为设计完整性证据。

## 8. 与 V1.0.1 主计划的关系

本 ADR 不修改 V1.0.1 计划报告（保持 frozen）。V1.0.1 主线 G1-G5 验收以 BF16 prefer 为唯一在 G2 ideal region 的候选，本 ADR 描述的 V1.2 路径是 stretch goal 之外的**潜在第二候选**入口，不影响 V1.0.0 / V1.1 的最终交付状态。
