# ADR-011 · V1.3 — Quantization-Aware Fine-Tuning (QAT) for INT8 ViT-L

> 状态：**Proposed**（V1.3 未来工作 / 论文方向设计文档，未实施）
> 触发：M1-progress 第二十五轮 ADR-010 § 5.3 — V1.2 ONNX-level Q/DQ stripping 与 V1.1 ModelOpt PyTorch-level disable_quantizer 等价（feat_layer_20 cos_min 0.9705 vs 0.971，speedup 2.39× vs 2.41×），DEFINITIVELY 闭合 V1.0.0+V1.1+V1.2 全部 mixed-precision 路径为 negative。
> 依据：第十四 / 十八 / 二十五轮三条 mixed-precision 路径联合证据指向**前段（blocks 0-15）累积的 INT8 量化噪声**是 root cause，末段 mixed-precision 不能 recover。
> 期望：QAT fine-tuning 在量化网格上重新优化模型权重，让前段 INT8 量化误差从 ~10⁻² 量级压到 ~10⁻³ 量级，跨过 G2 cos_min ≥ 0.99 阈值。

## 1. 决策点

如何让 SmoothQuant α=0.8 INT8 ONNX 的 feat_layer_20 cos_min 从 0.968 / 0.971 跨过 G2 阈值 0.99，同时保留 ≥ 2.2× speedup？

## 2. V1.0.0+V1.1+V1.2 mixed-precision 闭合证据链

| 路径 | 工具链层 | 实施轮次 | feat_layer_20 cos_min | b8 speedup | 结论 |
|---|---|---|---:|---:|---|
| Full SmoothQuant α=0.8 | PyTorch ModelOpt | V1.1 第十二轮 | 0.968 | 3.48× | speed ✅ / cos ❌ baseline |
| ModelOpt `disable_quantizer` skip 16-19 | PyTorch | V1.1 第十四轮 | 0.971 (+0.003) | 2.41× (-30%) | negative-ish |
| trtexec `--layerPrecisions=l16-19:fp32` | TRT command-line | V1.1 第十八轮 | 0.9683 (≈) | 3.43× (≈) | no-op (Q/DQ 边界即真精度边界) |
| **ONNX graph-level Q/DQ stripping** | ONNX library | **V1.2 第二十五轮** | **0.9705 (+0.003)** | **2.39× (-31%)** | **definitive negative — 与 V1.1 #2 等价** |

**机制（共同 root cause）**：ViT-L blocks 0-15 的 INT8 PTQ 累积量化误差通过 21 个 transformer block 传播，在到达 block 16 输入时已偏离 FP32 baseline 约 10⁻² 量级。即使 block 16-19 内部用 FP32 计算，输入数值已不正确，输出无法 recover。**改 block 16-19 内部精度只能让那 4 个 block 的 inference 更精确，但不能修复输入污染。**

要跨过 G2 cos_min ≥ 0.99 阈值（即 feat_layer_20 偏离 FP32 ≤ 10⁻²），必须在前段（blocks 0-15）就把量化误差压到 ~10⁻³ 量级。**PTQ + graph rewrite 已穷尽所有可能性，必须改用 QAT。**

## 3. 备选方案对比

| 方案 | 工作量 | 风险 | 期望 cos_min | 期望 speedup | 适用性 |
|---|---|---|---:|---:|---|
| **A** PTQ + 节点级 sensitivity（V1.1 已做） | 1-2 周 | 低 | 0.99+ for 单层量化 | 1.0-1.1× | speed 收益消失，已 negative |
| **B** SmoothQuant + α tuning（V1.1 已做） | 1 周 | 低 | 0.968 (best @ α=0.8) | 3.48× | speed 达标但 cos 差 0.022，已 negative |
| **C** Mixed-precision via PyTorch / TRT / ONNX（V1.1+V1.2 已做） | 1-2 周 | 低 | 0.97 (≈ skip 16-19) | 2.4× | 与 SmoothQuant + skip 等价，已 negative |
| **D** **QAT fine-tuning**（本 ADR） | **1-2 个月**（含数据集 + 训练 + 评估） | **中**（依赖训练资源 + ImageNet 数据集） | **0.99+ 期望** | **3.0-3.5× 期望** | **未实施 — V1.3 / 论文方向** |
| **E** 升级 TRT 到 11.x + Blackwell-native INT8 fusion | 数月（等版本） | 高（依赖 NVIDIA 上游） | 未知 | 未知 | 不可控时间线 |

## 4. 选择

**方案 D**：QAT 量化感知 fine-tuning，在 SmoothQuant Q/DQ ONNX 已就位的 calibration scaffolding 之上，把 PTQ → QAT。

## 5. 实施约束

### 5.1 数据集要求

QAT fine-tuning 必须在与下游任务（dense prediction）一致的 distribution 上做。当前选项：

| 数据集 | 状态 | 用途 |
|---|---|---|
| ImageNet val 50K | **HF gated repo 403 阻塞**（外部 blocker） | 主选；与 DINOv3 预训练 LVD-1689M distribution 最近 |
| Imagenette2-320 val 1000 张 | 已可用（V1.1 主路径） | 数据量太小，QAT 需要 ≥ 10K 样本，不够用 |
| 自建 dense prediction 数据集（depth/segmentation） | 未启动 | 论文方向，与 DINOv3 下游任务一致但工作量大 |

**判定门槛**：QAT 实施前必须 unblock 至少一个 ≥ 10K 张样本的数据集。

### 5.2 框架与工具链

| 选型 | 候选 | 推荐 |
|---|---|---|
| QAT 框架 | NVIDIA Model Optimizer (`modelopt.torch.quantization` QAT mode) / PyTorch FX QAT / Brevitas | **ModelOpt QAT**（与 V1.1 PTQ 同框架，导出 Q/DQ ONNX 直接复用 V1.0.0/V1.1/V1.2 整套 trtexec build pipeline） |
| 训练硬件 | RTX 5080（16 GB VRAM）/ A100 / H100 | RTX 5080 batch 8 ViT-L QAT 估算 ≥ 24 小时；优先在 A100/H100 节点跑 |
| 训练数据 | ImageNet val 50K 或更大 | 必须 ImageNet 级别 distribution |
| 监督信号 | DINOv3 self-supervised loss / DPT 风格下游任务 loss | DINOv3 loss 复杂度高（teacher-student），DPT 下游任务简单但需要标注数据 |
| 训练步数 | 1-5 epoch fine-tune | PTQ 已是 init，不需要从头训练 |

### 5.3 期望结果矩阵

| 指标 | V1.1 PTQ baseline | V1.3 QAT 目标 | V1.0.1 G2 阈值 |
|---|---:|---:|---:|
| feat_layer_20 cos_min（vs FP32） | 0.968 | **≥ 0.99 期望** | ≥ 0.99 |
| feat_layer_20 cos_mean | 0.982 | **≥ 0.995 期望** | — |
| trtexec b8 speedup vs FP32 | 3.48× | **≥ 3.0× 期望**（QAT 不改变 graph，只改 weight 数值） | ≥ 2.2× |
| 训练成本 | 0（PTQ-only） | **1-5 GPU-day** | — |

如果实测 cos_min ≥ 0.99 同时 speedup ≥ 3.0×，**这就是首个完整满足 G2 ideal region 的 INT8 候选**，与 BF16 prefer 平起平坐（甚至速度更高）。

## 6. 实施步骤（建议）

### 6.1 工程层

1. 新模块 `Code/src/dinov3_trt/quantization/qat.py`：
   - `make_qat_model(model, *, calibration_loader)`：包装 PyTorch HF DINOv3 model，调用 `modelopt.torch.quantization.quantize` 用 INT8_DEFAULT_CFG（与 V1.1 SmoothQuant 不同 — QAT 是从头量化所有层），返回 fake-quantized model。
   - `train_qat_step(model, batch, optimizer, loss_fn)`：单步训练函数，自动处理 STE（straight-through estimator）梯度传播。
   - `export_qat_to_onnx(model, output_path, *, contract)`：调用 `torch.onnx.export` 导出 QAT 训完的 Q/DQ ONNX，与 V1.1 ModelOpt PTQ 输出格式兼容。
2. 新驱动 `Code/scripts/train_qat_dinov3.py`：
   - 加载 ImageNet（unblock 后）或下游数据集。
   - 调用 `make_qat_model` + 训练 N epoch。
   - 调用 `export_qat_to_onnx` 写出 ONNX，文件名规范：`dinov3_vitl16_4out.qat.<dataset>.<epochs>e.onnx`。
3. 新测试 `Code/tests/test_qat.py`：
   - 用 mini synthetic model + 1 个 batch 验证 `make_qat_model` 返回 fake-quantized 节点结构正确。
   - 用 mini batch 跑 1 步 train/eval round-trip，验证 STE 梯度。
   - 不依赖真实 DINOv3 权重（保持本地可跑）。

### 6.2 远端实验

复用 V1.0.0/V1.1/V1.2 整套 pipeline：

1. QAT 训完后 → `dinov3_vitl16_4out.qat.imagenet.3e.onnx`（与 SmoothQuant ONNX 同目录、同命名风格）。
2. `trtexec --int8` build engine（与 V1.1 SmoothQuant build 流程一致）。
3. `benchmark_trtexec.py` locked 2752 MHz + spin-wait benchmark。
4. `evaluate_engine_pair_on_images.py` Imagenette eval（或 unblock 后的 ImageNet val）cosine。
5. `summarize_trtexec_benchmarks.py` + `build_benchmark_matrix.py` + `build_int8_tradeoff_figure.py`：matrix +2 行 / tradeoff scatter +1 点。

## 7. 风险与已知未知

| 风险 | 缓解 |
|---|---|
| **训练发散 / NaN**：ViT-L 24 个 transformer block 用 STE 反向传播时梯度可能爆炸 | 用 ModelOpt QAT 默认配置（已经在 ResNet/ViT 上验证过 STE 稳定性）；如不稳定可用更小 learning rate 或 partial layer freezing |
| **训练成本超预算**：RTX 5080 16 GB 不够 ViT-L QAT，需要 A100/H100 节点 | V1.3 实施前必须确认训练资源；如只有 RTX 5080，缩 batch + gradient accumulation 训 5+ 天 |
| **ImageNet 仍 blocked**：QAT 需要 ImageNet-级 distribution，外部 blocker 不解决就启动不了 | unblock 之前不实施；或考虑 LAION-OpenImages 等替代数据集（需对照 DINOv3 LVD distribution） |
| **QAT 后 cos 仍未到 0.99**：可能 SmoothQuant α=0.8 的初始化不够好，QAT 也救不回来 | 备用方案：从 BF16 fine-tune 开始（即 BF16 → QAT INT8）；或扩大 α sweep 找更好初始化 |
| **TRT 10.13 上 QAT ONNX 与 SmoothQuant ONNX 行为不一致**：Myelin 优化器对训练时的 fake-quant 节点选择不同 tactic | 实测验证；必要时 lock TRT 10.16.1（V1.0.1 ADR-008 理想版本） |

## 8. 实施判定门槛

V1.3 方案 D 应在以下条件**全部**满足时启动：

1. **数据集 unblock**：ImageNet val 50K 或同等 distribution 的 ≥ 10K 样本数据集可用。
2. **训练资源到位**：A100/H100 GPU-day 预算 ≥ 5 days，或 RTX 5080 GPU-week ≥ 1 week。
3. **专项时间预算**：1-2 个月专项工程 + 论文写作时间（本期 V1.0.0 + V1.1 + V1.2 共耗 ~6 周）。
4. **下游任务 baseline 就位**：至少一个下游任务（depth / segmentation）的 FP32 baseline + DPT 头训练 pipeline，让 QAT 的 cosine 可以转成下游任务 mAP/AP@IoU 等指标，避免"cos 高了但下游精度没改善"的 false positive。

否则保留为论文 future work / 后续阶段任务，引用本 ADR 作为方向完整性证据。

## 9. 与 V1.0.0 / V1.1 / V1.2 主计划的关系

本 ADR 不修改 V1.0.1 计划报告（保持 frozen）。V1.0.1 主线 G1-G5 验收以 BF16 prefer 为唯一在 G2 ideal region 的候选；V1.1+V1.2 stretch goals 全部 negative 闭合。本 ADR 描述的 V1.3 路径是 V1.0.0+V1.1+V1.2 之外的**潜在第二候选入口**，不影响当前最终交付状态。

## 10. 与 ADR-010 的关系

ADR-010（V1.2 ONNX Q/DQ stripping）的状态从 Proposed 升级到 Implemented · Negative result（第二十五轮）。ADR-011（本文档）是 ADR-010 negative result 的自然延续：ADR-010 § 5.3 明确指向"V1.3 / 论文阶段的研究问题（QAT、量化感知 fine-tuning）"，本 ADR 把那一句指向落地为完整设计文档。
