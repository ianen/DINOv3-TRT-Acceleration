# ADR-012 · V1.0.2 — CUDA Graphs + Pinned Host Memory（Quick Win #1）

> **状态**：**Implemented-Partial**（V1.0.2 Week 1 完成。CUDA Graphs 已就位 + 实测；Pinned Memory staging 推迟到 V1.0.2 第二切以避免破坏 caller buffer 契约。）
> **闭合时间**：2026-05-02
> **实测速度增益**（real RTX 5080 + TRT 10.13.2.6）：
> - r224 b1 cpp end-to-end median **1.135×**（3.33 ms → 2.94 ms）⭐ 跨过 ADR §5 G1 ≥ 1.10 目标
> - r518 b8 cpp end-to-end median ~1.005×（compute-dominated，符合"high batch launch overhead 占比小"假说）
> - 跨语言 parity（C++ Graph 路径 vs Python TRT runtime）：4 layer outputs 全部 `max_abs_error=0` / `cosine_similarity=1.0` — bit-exact ✓
> - `outputs_all_finite=true`，无 NaN
>
> **历史档案 - 原 Proposed 状态**：
> **触发**：V1.0.1 codebase audit 发现 `Code/cpp/src/trt_inferer.cpp` 是**单 stream 顺序**（`cudaMemcpyAsync(H2D) → enqueueV3() → cudaMemcpyAsync(D2H) → cudaStreamSynchronize()`），无 CUDA Graph 捕获重放，无 pinned host memory。
> **依据**：CUDA Graphs 在 ViT-L/16 这种"固定 shape + 高频重复 launch"的负载下能消除每次推理 50–200 μs 的 launch overhead；pinned memory 让 H2D/D2H 拷贝可与 GPU compute 重叠，PCIe transfer 不再是 bottleneck。
> **期望**：bit-exact 改动（不动数值精度），增益 ~1.10× 复合（low-batch 1.05–1.15× + 全配置 1.05–1.15×），cos_min 完全保留 ≥ 0.997 V1.0.1 baseline。
> **决策点**：是否在 V1.0.2 Week 1–2 同时引入 CUDA Graphs 和 Pinned Memory？为何不分两个 ADR？

---

## 1. 决策点

V1.0.1 的 C++ runtime（`TRTInferer::Impl`）是单 stream 顺序推理，存在两个独立但**互补**的优化机会：

1. **launch overhead** — 每次 inference 调 `cudaMemcpyAsync(H2D) + enqueueV3() + cudaMemcpyAsync(D2H)` 共 3 个 API call，每个产生约 5–20 μs 的 host-side overhead；低 batch 时累积可达 50–200 μs，相对 r518 b1 的 ~10 ms 推理已是 1–2% 占比。
2. **PCIe transfer** — 当前 H2D/D2H 用 pageable host memory，CUDA driver 必须先做 DMA staging，效率比 pinned memory 低 30–50%；且 pageable 不能与 kernel 重叠。

两者**联合实施**比分开做更好：CUDA Graphs 捕获时必须包含完整的 H2D-compute-D2H sequence；如果 H2D 用 pageable 后再切 pinned，Graph 必须重 capture。同 ADR 一次到位。

---

## 2. 现状（V1.0.1 frozen，single-stream）

`Code/cpp/src/trt_inferer.cpp` 关键代码路径（line 268-298）：

```cpp
// V1.0.1 当前实现（伪代码简化）
cudaMemcpyAsync(input_buffer_.device_ptr, host_input,         // H2D
                input_size, cudaMemcpyHostToDevice, stream_);
context_->enqueueV3(stream_);                                  // compute
for (auto& output_buffer : output_buffers_) {
    cudaMemcpyAsync(host_output, output_buffer.device_ptr,    // D2H
                    output_size, cudaMemcpyDeviceToHost, stream_);
}
cudaStreamSynchronize(stream_);                                // wait
```

问题：
- 3 个 `cudaMemcpyAsync` + 1 `enqueueV3` = **4 个独立 launch**，每次都付 host-side overhead
- `host_input` / `host_output` 是 pageable，driver 需双缓冲拷贝
- `cudaStreamSynchronize` 是粗粒度等待，没法 overlap 下一帧

## 3. 备选方案对比

| 方案 | 工作量 | 增益 | 风险 | bit-exact |
|---|---|---|---|---|
| **A** 仅 pinned memory（不动 Graph 路径）| 0.5 周 | 1.05–1.15× across configs | 低 | ✅ |
| **B** 仅 CUDA Graphs（仍 pageable 拷贝）| 1 周 | 1.05–1.15× at b≤4，b≥16 negligible | 中（dynamic shape）| ✅ |
| **C** **CUDA Graphs + Pinned Memory（本 ADR）** | **1 周** | **复合 ~1.10×** | 中 | ✅ |
| **D** 等 V1.0.2 Week 3 multi-stream 顺便做 | 与 ADR-015 合并 | 不正交，难单独验证 | 中 | ✅ |
| **E** 重写 inference loop 用 cuStream API（low-level）| 2 周 | 略高但回报递减 | 高 | ✅ |

**选择**：**方案 C** — CUDA Graphs + Pinned Memory 同 ADR 实施。理由：
1. 两者技术上耦合（Graph 必须 capture 包含 H2D 路径，pinned 提升此路径效率）
2. 都是 bit-exact，不引入 cos 风险，可作为后续 sparsity / FP8 工作的稳定 V1.0.2 基线
3. 工作量合理（1 周），ROI 明确（复合 ~1.10× 在所有 V1.0.1 benchmark 点上叠加）

## 4. 实施设计

### 4.1 新文件

#### `Code/cpp/include/dinov3_trt/pinned_buffer.h`

RAII 包装 pinned host memory：

```cpp
namespace dinov3_trt {

class PinnedBuffer {
public:
    explicit PinnedBuffer(size_t bytes);
    ~PinnedBuffer();

    PinnedBuffer(const PinnedBuffer&) = delete;
    PinnedBuffer& operator=(const PinnedBuffer&) = delete;
    PinnedBuffer(PinnedBuffer&& other) noexcept;
    PinnedBuffer& operator=(PinnedBuffer&& other) noexcept;

    void* host_ptr() const noexcept { return host_ptr_; }
    size_t size_bytes() const noexcept { return bytes_; }

private:
    void* host_ptr_{nullptr};
    size_t bytes_{0};
};

}  // namespace dinov3_trt
```

实现核心：`cudaMallocHost(&host_ptr_, bytes)` 替代 `malloc`；析构 `cudaFreeHost`；move semantics 标准 RAII pattern。

#### `Code/cpp/include/dinov3_trt/cuda_graph_pool.h`

按 (batch_size, resolution) 索引的 Graph cache：

```cpp
namespace dinov3_trt {

struct GraphKey {
    int batch_size;
    int resolution;
    bool operator==(const GraphKey&) const = default;
};

class CudaGraphPool {
public:
    explicit CudaGraphPool(size_t max_cached = 6);  // 2 res × 3 batch sizes

    // Returns a pre-captured graph for the given key; null if not cached.
    // Caller invokes cudaGraphLaunch + cudaStreamSynchronize.
    cudaGraphExec_t get(const GraphKey& key) const;

    // Capture a new graph from the given lambda (which performs the full
    // H2D + enqueueV3 + D2H sequence on the captured stream).
    cudaGraphExec_t insert(const GraphKey& key,
                            cudaStream_t capture_stream,
                            std::function<void(cudaStream_t)> capture_fn);

    void clear();

private:
    size_t max_cached_;
    // implementation: LRU map with cudaGraphExec_t handles
};

}  // namespace dinov3_trt
```

### 4.2 修改 `Code/cpp/src/trt_inferer.cpp`

集成两个新组件，feature flag 默认开启但可关：

```cpp
// V1.0.2 集成后（伪代码）
class TRTInferer::Impl {
    cudaStream_t stream_;
    PinnedBuffer pinned_input_;          // ← 替代 malloc 的 host_input
    std::vector<PinnedBuffer> pinned_outputs_;
    CudaGraphPool graph_pool_;            // ← 新增
    bool use_cuda_graph_{true};            // ← env var DINOV3_USE_CUDA_GRAPH 控制
};

void TRTInferer::Impl::infer(const float* host_input_data, ...) {
    GraphKey key{batch_size, resolution};

    // 复制 user 数据到 pinned buffer（一次性，pinned 后续与 GPU 重叠）
    std::memcpy(pinned_input_.host_ptr(), host_input_data, ...);

    if (use_cuda_graph_) {
        cudaGraphExec_t exec = graph_pool_.get(key);
        if (!exec) {
            // 第一次见到此 (batch, res) → capture
            cudaStreamBeginCapture(stream_, cudaStreamCaptureModeGlobal);
            do_h2d_compute_d2h_sequence(stream_);
            cudaGraph_t g;
            cudaStreamEndCapture(stream_, &g);
            exec = graph_pool_.insert(key, stream_, [&](cudaStream_t s){
                do_h2d_compute_d2h_sequence(s);
            });
        }
        cudaGraphLaunch(exec, stream_);
    } else {
        // fallback: V1.0.1 顺序路径
        do_h2d_compute_d2h_sequence(stream_);
    }
    cudaStreamSynchronize(stream_);
}
```

### 4.3 Feature flag

| 环境变量 | 默认值 | 含义 |
|---|---|---|
| `DINOV3_USE_CUDA_GRAPH` | `1` | 启用 CUDA Graph capture/replay；`0` fallback 到 V1.0.1 路径 |
| `DINOV3_USE_PINNED_MEMORY` | `1` | 启用 pinned host memory；`0` fallback 到 pageable malloc |
| `DINOV3_CUDA_GRAPH_CACHE_SIZE` | `6` | LRU cache 大小（可调）|

## 5. 验收标准（acceptance gate）

| 指标 | V1.0.1 baseline | V1.0.2 ADR-012 后阈值 |
|---|---|---|
| **G1 r518 b1 cpp end-to-end speedup** | 1.0× ref | **≥ 1.10×**（最低）/ **≥ 1.15×**（target）|
| **r224 b8 cpp end-to-end speedup** | 1.0× ref | **≥ 1.05×** |
| **r518 b8 cpp end-to-end speedup** | 1.0× ref（在 V1.0.1 BF16 prefer 3.40× 之上）| **≥ 1.05×**（即新总速 ≥ 3.57×）|
| **BF16 prefer cos_min on real ImageNet val 1000** | 0.9977 | **≥ 0.997**（bit-exact 应等于 baseline）|
| pytest 357 → 357+N | passed | **passed**，N ≥ 4 新测 |
| ruff + mypy --strict | 全绿 | **全绿** |

**关键 gate**：cos_min 必须**完全等于** V1.0.1 baseline（0.9977，不退化任何小数位）— CUDA Graphs / pinned memory 是 bit-exact 改动，任何 cos 退化说明实现有 bug。

## 6. 风险与缓解

| 风险 | Likelihood | Impact | 缓解 |
|---|---|---|---|
| CUDA Graphs 不兼容动态 shape | **High** | High | 每 (batch, res) 独立 capture；cache 6 个；fallback 路径保留（feature flag）|
| Graph capture 失败（memcpy 路径有 host-side branch）| Medium | Medium | capture mode 用 `cudaStreamCaptureModeRelaxed`；问题层 fallback 到 ad-hoc launch |
| Pinned memory 分配失败（OS 限制 ~1/8 RAM）| Low | Medium | 对大 batch（r518 b32）单 buffer 约 80 MB，远低于限制；预分配 + 错误处理 |
| feature flag 默认开启导致回归 | Low | High | CI 跑 N 次（DINOV3_USE_CUDA_GRAPH=0 vs 1）双 mode；任一退化 block PR |
| 多 buffer pinned 在 multi-stream 引入竞争（ADR-015 未做时）| Low | Low | 当前单 stream，无问题；ADR-015 时再处理 |
