# ADR-014 · V1.0.2 — TensorRT 10.13.2.6 → 10.16.1 升级验证（前置 milestone）

> **状态**：**Proposed**（V1.0.2 Week 0 前置工作，未实施）
> **触发**：V1.0.2 ADR-016（2:4 structured sparsity）依赖 TRT ≥ 10.16 成熟的 sparse Tensor Core kernel；ADR-017（FP8 refined scaling）在 Blackwell sm_120 上需较新 TRT FP8 kernel 调优。当前生产版本 TRT 10.13.2.6。
> **依据**：CLAUDE.md 项目计划 §"技术栈"明确"TensorRT ≥ 10.13（理想 10.16.1；10.8 仅作 baseline 对照）"；V1.0.1 frozen baseline 全部基于 10.13.2.6 实测。
> **期望**：升级到 10.16.1 不退化 V1.0.1 任何已闭合的 cos / speedup 数字（容差 ±2%），并解锁 V1.0.2 sparsity / FP8 路径。
> **决策点**：是否在 V1.0.2 启动前执行 TRT 升级？升级后如何防止 V1.0.1 论文 claims 失效？

---

## 1. 决策点

V1.0.2 多个 ADR（016 sparsity / 017 FP8 / 018 custom kernel）依赖较新 TRT 版本的成熟 kernel。TRT 10.13.2.6 → 10.16.1 升级是否必要？如何在不破坏 V1.0.1 frozen baseline 的前提下完成升级？

---

## 2. 当前 TRT 10.13.2.6 vs 目标 10.16.1 的差异

| 维度 | TRT 10.13.2.6（V1.0.1 当前）| TRT 10.16.1（V1.0.2 目标）| Delta |
|---|---|---|---|
| 发布时间 | 2026 早期 | 2026 中期 | ~3-4 个月 |
| Blackwell sm_120 支持 | 基础（部分 ViT 算子未 fully tuned）| 成熟（含 ViT GELU/LayerNorm fusion 优化）| 中等 |
| 2:4 structured sparsity kernel | 实验性（kernel selection 可能 fallback dense）| 生产级（sparse Tensor Core 自动 dispatch）| **关键差异**（V1.0.2 ADR-016 阻塞依赖）|
| FP8 kernel | Hopper-style 但 Blackwell 调优有限 | Blackwell-native FP8 path | 中等（V1.0.2 ADR-017 受益）|
| CUDA Graphs API | 完全支持 | 完全支持 | 无差异 |
| Q/DQ INT8 calibrator | 完全支持 | 完全支持（API 无 break）| 无差异 |
| BF16 / FP16 builder | 完全支持 | 完全支持，但 kernel 可能微调 | **风险点**（V1.0.1 numbers 可能微变）|
| Engine binary 兼容性 | — | **不兼容**：需重新 build engine | 工作量 |

**结论**：升级**必要但不紧急** — V1.0.2 quick wins（CUDA Graphs / pinned memory / multi-stream）不依赖 TRT 升级，可在 10.13.2.6 上做；升级真正解锁的是 ADR-016 sparsity（Week 5–6）+ ADR-017 FP8（Week 7–8）+ ADR-018 custom kernel（Week 9–12）。

---

## 3. 升级风险与缓解

| 风险 | Likelihood | Impact | 缓解 |
|---|---|---|---|
| BF16 prefer cos_min 退化（< 0.997） | Low-Medium | **High**（破坏 V1.0.1 论文 claims）| Week 0 升级后立即跑 orchestrator 验证；不达标 stay on 10.13.2.6 |
| BF16 prefer speedup 退化（< 3.40× cpp r518 b8）| Medium | High | 同上，benchmark matrix 复现验证 |
| INT8 SmoothQuant α=0.8 cos_min 进一步下降 | Low | Medium | 已是 R2 应急候选，再降也不影响主交付（BF16 prefer） |
| Engine binary 不兼容需全部重建 | Certain | Low | 工作量约 2-3 hour（45+ engines），timing cache 加速 |
| Python `tensorrt` 包 ABI 变更 | Low | Medium | 测试 `from kaggle.api... import KaggleApi` 类似 import 通过 |
| C++ runtime 编译 break | Low | Medium | TRT 10.x 系列 API 稳定；header `NvInfer.h` 接口不变 |
| Windows ↔ Linux 工具链差异 | Low | Low | 项目仅 Windows + RTX 5080；无跨平台 |
| 升级失败 + 卸载困难 | Low | High | 升级前完整备份 `C:\Program Files\NVIDIA GPU Computing Toolkit\TensorRT-10.13.2.6\`，可 rollback |

## 4. 备选方案对比

| 方案 | 工作量 | 风险 | V1.0.2 阻塞解锁 | V1.0.1 baseline 保护 |
|---|---|---|---|---|
| **A** 立即升级到 10.16.1，重建所有 engines + 复现 V1.0.1 数字 | 1 周（含 validation） | 中（baseline 可能微变 ±2%） | ✅ 全部解锁 | 风险：±2% 容差内可接受；超容差 rollback |
| **B** 双版本并存（10.13.2.6 + 10.16.1），V1.0.1 走旧版，V1.0.2 走新版 | 2 周（环境管理复杂） | 低（baseline 无变化） | ✅ 全部解锁 | ✅ 完整保护 |
| **C** 不升级，V1.0.2 在 10.13.2.6 上做 sparsity（kernel fallback 风险）| 0 | 高（sparsity 不达标）| ⚠️ 部分（sparsity / FP8 受限） | ✅ 完整保护 |
| **D** 等 NVIDIA 发布 10.17 / 11.0 再升级 | ∞（不可控） | 不可控 | ❌ 阻塞 V1.0.2 sparsity | ✅ 完整保护 |

**选择**：**方案 A** — 立即升级。理由：
1. V1.0.2 ADR-016/017 强依赖较新 TRT，方案 C 风险过高
2. 方案 B 双版本并存增加 ssh 远程管理复杂度，性价比低（V1.0.1 已 frozen，不会再改 baseline）
3. ±2% 容差是合理工程容忍度（论文 claims 用峰值数字 3.86× / 3.40×，±2% 是 3.78–3.94× 区间，仍可表述为 "≈ 3.86×"）

## 5. 实施步骤（Week 0 详细 checklist）

### 5.1 升级前备份（Day 1, ~1 hour）

```bash
# 远端 Windows
ssh windows-pc

# 备份当前 TRT 10.13.2.6 安装
cd /d "C:\Program Files\NVIDIA GPU Computing Toolkit"
xcopy /E /I /H /Y TensorRT-10.13.2.6 TensorRT-10.13.2.6.backup-2026-05-02

# 备份 venv 中的 TRT Python wheel
cd /d D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code
.venv\Scripts\python.exe -m pip freeze | findstr tensorrt > Artifacts\trt_freeze_pre_upgrade.txt
copy .venv\Lib\site-packages\tensorrt-10.13.2.6.dist-info Artifacts\trt_dist_info_pre_upgrade

# 备份所有当前 engines（可重建但 timing cache 重用）
xcopy /E /I /Y Artifacts\engines Artifacts\engines.backup-pre-trt-10.16.1
xcopy /E /I /Y Artifacts\timing_cache Artifacts\timing_cache.backup-pre-trt-10.16.1
```

### 5.2 下载并安装 TRT 10.16.1（Day 1, ~30 min）

```powershell
# 1. 从 NVIDIA Developer 下载 TensorRT 10.16.1 GA Windows zip
#    https://developer.nvidia.com/tensorrt/download
#    选择 CUDA 12.x 对应版本（cu128 兼容）

# 2. 解压到 C:\Program Files\NVIDIA GPU Computing Toolkit\TensorRT-10.16.1

# 3. 更新 PATH 环境变量（如需要）：
#    旧：%PATH%;C:\Program Files\...\TensorRT-10.13.2.6\bin;...\TensorRT-10.13.2.6\lib
#    新：%PATH%;C:\Program Files\...\TensorRT-10.16.1\bin;...\TensorRT-10.16.1\lib

# 4. 安装 Python wheel
cd /d D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code
.venv\Scripts\python.exe -m pip uninstall -y tensorrt
.venv\Scripts\python.exe -m pip install "C:\Program Files\NVIDIA GPU Computing Toolkit\TensorRT-10.16.1\python\tensorrt-10.16.1-cp310-none-win_amd64.whl"
```

### 5.3 import + 版本验证（Day 1, ~5 min）

```bash
ssh windows-pc 'cd /d D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code && .venv\Scripts\python.exe -c "import tensorrt as trt; print(trt.__version__); print(trt.Builder(trt.Logger()).platform_has_fast_fp16, trt.Builder(trt.Logger()).platform_has_fast_int8)"'
```

期望输出：
```

True True
```

### 5.4 重建 V1.0.1 关键 engines（Day 1–2, ~3 hour）

仅重建主交付候选 + R2 应急候选（不必重建所有 45 个 engine）：

```bash
ssh windows-pc 'cd /d D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code && \
    .venv\Scripts\python.exe scripts\build_engine_trtexec.py \
        --onnx Artifacts\onnx\dinov3_vitl16_4out.onnx \
```

### 5.5 V1.0.1 数字复现验证（Day 2, ~2 hour）

```bash
# 1. 跑 V1.0.1 第 67 轮 orchestrator on real ImageNet val 1000
ssh windows-pc 'cd /d D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code && \
    .venv\Scripts\python.exe scripts\run_imagenet_val_post_download.py \
        --engines-dir Artifacts\engines\v102 \
        --report-dir Artifacts\reports\v102_baseline_validation'

# 期望输出（容差 ±2%）：
#   bf16_prefer:                cos_min ≥ 0.978 (target ≥ 0.997 ideal)
#   int8_smoothquant_a080:      cos_min ≥ 0.952 (target ≥ 0.97 ideal)

# 2. 跑 trtexec benchmark 关键点

# 期望：speedup ≥ 3.78× （3.86× × 0.98 容差）

# 3. 跑 cpp end-to-end benchmark
```

### 5.6 acceptance criteria（gate）

| 指标 | V1.0.1 baseline | TRT 10.16.1 后阈值 | gate verdict |
|---|---|---|---|
| BF16 prefer cos_min on real ImageNet val 1000 | 0.9977 | **≥ 0.997**（容差 -0.0007）| 必须达成，否则 rollback |
| BF16 prefer trtexec speedup r518 b8 | 3.86× | **≥ 3.78×**（容差 -2%）| 必须达成 |
| BF16 prefer cpp end-to-end r518 b8 | 3.40× | **≥ 3.33×**（容差 -2%）| 必须达成 |
| INT8 SmoothQuant α=0.8 cos_min | 0.9727 | **≥ 0.95**（R2 仍通过）| 容许 -0.022 退化 |
| pytest 357 passed | 357/360 | **357/360** | 必须达成 |
| 全量 ruff + mypy --strict | 全绿 | **全绿** | 必须达成 |

## 6. Rollback 计划

若 §5.6 任何 must-达成项不通过：

1. **保留 TRT 10.16.1 安装** 但**不**让项目使用：
   ```bash
   ssh windows-pc 'cd /d D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code && \
       .venv\Scripts\python.exe -m pip uninstall -y tensorrt && \
       .venv\Scripts\python.exe -m pip install "C:\Program Files\...\TensorRT-10.13.2.6.backup-...\python\tensorrt-10.13.2.6-cp310-none-win_amd64.whl"'
   ```
2. **修改 PATH** 让 trtexec.exe 指回 10.13.2.6
3. **恢复 engines** from `Artifacts\engines.backup-pre-trt-10.16.1`
4. **更新 V1.0.2 plan**：标记 ADR-014 为 **Rejected**，ADR-016 sparsity 改为 stretch / 待 NVIDIA 出 10.17+ 再启动
5. **V1.0.2 quick wins（ADR-012 / 013）继续**，无依赖 TRT 升级
