# ADR-018 · V1.0.2 — Custom CUDA Fused Attention Pre-projection Kernel（Stretch）

> **状态**：**Gated-PASS-but-Limited-ROI**（V1.0.2 第 95+ 轮，2026-05-02 profiler 实证）
>
> **trtexec --exportProfile 实测 on r518 b8 BF16 prefer engine + RTX 5080 + TRT 10.13.2.6**:
>
> | 类别 | 累计 averageMs | 占比 | Notes |
> |---|---|---|---|
> | Total GPU compute | 8.862 ms | 100% | trtexec --useSpinWait baseline |
> | **Attention 层** | 2.343 ms | **26.4%** | ✅ ADR §1 gating ≥ 20% PASSES |

>
> **ROI 重新估算**（基于实证 profile，非 ADR plan-time 估算）：
> - 假设 custom kernel 把 attention 26.4% 部分减少 25-30% kernel time
> - **总加速 = 26.4% × 0.25 = 6.6% overall**（vs ADR plan-time 估 10-30%）
> - 即使最佳情况 30% reduction：**总加速 7.9% overall**
> - **远低于 ADR plan-time 高端估算**
>
> **ADR §1 gating 决策**:
> - ✅ Profiler 显示 attention ≥ 20% unfused（实测 26.4%）→ Gating PASSES
> - ❌ ROI 实证 ~7% << ADR plan-time 估 10-30%
> - 4 周研究级工作（含 NaN risk + cuDNN/Triton 探索） vs ~7% 单点 speedup
> - **决策**：状态留为 Gated-PASS-but-Limited-ROI，不立即触发实施
>
> **V1.0.2 envelope 重新评估**（含所有未触发的 stretch ADRs 全部完美工作的乐观估算）:
>
> | 路径 | 单步增益 | 累计 r518 b8 |
> |---|---|---|
> | V1.0.1 baseline | — | 3.40× |
> | + ADR-012 CUDA Graphs (实测 1.005×) | 1.005 | 3.42× |
> | + ADR-014 TRT 10.16.1 upgrade (5-10% est) | 1.075 | 3.67× |
> | + ADR-018 custom kernel (profile-based 7% real) | 1.07 | **3.93×** |
> | **5.0× 旗舰目标 gap:** | | **1.27×** (27% 仍缺) |
>
> **决定性 conclusion**：**V1.0.2 5.0× r518 b8 在不依赖 V1.3 QAT 时数学上不可达**。即使触发所有 user-blocked ADRs（014）+ 完整研究级 custom kernel（018），envelope 上限 ≈ 3.93-4.20×。
>
> **历史档案 - 原 Proposed 状态**：
> **触发**：V1.0.2 Week 1–8 quick wins 完成后，如 nsys / nsight-compute profiler 显示 attention pre-projection（LayerNorm + RoPE + QKV proj）路径仍占 inference 时间 ≥ 20% 且未被 TRT 自动 fuse，则手 roll 一个 fused kernel + TRT plugin 接入。
> **依据**：FlashAttention 系列证明 attention 内部融合能拿 1.5–2× kernel-level 增益；DINOv3 RoPE 实现非标准（V1.0.0 ADR-007 改造），TRT 的内置 attention fuser 可能因 RoPE 处理而 fail to fuse。
> **期望**：1.10–1.30× speedup on attention-bound 层；高复杂度高风险（NaN under bf16 / sm_120 register pressure）；feature flag default off + per-commit cos gate ≥ 0.99；**研究级**项，不是 V1.0.2 必交付。
> **决策点**：是否值得投 4 周做 custom kernel？profiler 不显示 attention 是 hot 怎么办？

## 1. 决策点（gating 条件）

本 ADR 是 V1.0.2 **stretch 项**，仅在以下条件满足时启动：

1. **Week 1–8 完成后**：ADR-012/013/015/016/017 全部到位（即使部分 Rejected）
2. **profiler 实证**：用 `nsys profile` + `ncu` 分析 latest BF16 prefer + sparsity engine on r518 b8，attention pre-projection（block.X.norm1 → block.X.attn.qkv → block.X.attn.rope）路径占 ≥ 20% kernel time
3. **TRT 自动 fusion 失败**：通过 trtexec `--profilingVerbosity=detailed` 输出确认 RoPE 节点未被 fuser 处理（保留为独立 kernel）
4. **时间预算**：V1.0.2 总周期还剩 ≥ 4 周

任一不满足 → 本 ADR 状态 **Deferred**，留 V1.0.3 / paper future work。

---

## 2. 现状 / 假设瓶颈

DINOv3 ViT-L 每个 transformer block 的 attention 前向：

```
input -> LayerNorm -> linear(qkv) -> reshape -> RoPE rotate(q, k) -> attention -> ...
```

V1.0.0 ADR-007 已用源码改造消除 RoPE 的 `aten::if` 条件分支，但 RoPE 仍是独立的 element-wise rotation 节点。TRT 10.13.2.6 的 attention fuser 不一定能跨 RoPE 边界融合。

**假设**：profiler 显示每 block 有 4–5 个独立 kernel launch（norm / qkv proj / rope_q / rope_k / attention），而理论上前 4 个能 fuse 为单 kernel。24 layer × 4 个减少的 kernel launch = 96 个 launch overhead 节省。

---

## 3. 备选方案对比

| 方案 | 工作量 | speedup | 风险 | 维护成本 |
|---|---|---|---|---|
| **A** 不做（接受 V1.0.2 Week 1–8 增益）| 0 | baseline | 0 | 0 |
| **B** **手 roll 单 kernel，TRT plugin 接入（本 ADR）** | **4 周** | **1.10–1.30× attention-bound 层** | **高** | 高（CUDA 代码维护）|
| **C** 用 cuDNN 9.x 的 fused attention API | 1 周 | 类似 | 低 | 低 | 
| **D** 等 TRT 11 自动 fuse | ∞（不可控） | 未知 | 0 | 0 |
| **E** Triton-based kernel（Python-friendly）| 3 周 | 类似但 deployment 复杂 | 中 | 中 |

**选择**（如启动）：**方案 C 优先 / B 备选**。

cuDNN 9.x 的 Fused Multi-Head Attention API 是 NVIDIA 官方维护，可信度高。如 cuDNN 接口能覆盖 RoPE 变体（推测能，因 RoPE 现已加入 PyTorch 标准 SDP attention path）则用 C；否则 fallback 手 roll B。

## 4. 实施设计（方案 B 详细，方案 C 类似但更简）

### 4.1 新文件 `Code/cpp/kernels/fused_attention_pre.cu`

```cuda
// Fused: LayerNorm + linear(qkv) + RoPE(q, k)
// Input:  [B, T, C]  (B=batch, T=tokens=197, C=hidden=1024)
// Output: q, k, v  each [B, H, T, D]  (H=heads=16, D=64)
//
// Memory-bound; uses shared memory + register tiling
__global__ void fused_attn_pre_kernel(
    const __nv_bfloat16* __restrict__ input,        // [B, T, C]
    const __nv_bfloat16* __restrict__ ln_weight,    // [C]
    const __nv_bfloat16* __restrict__ ln_bias,      // [C]
    const __nv_bfloat16* __restrict__ qkv_weight,   // [C, 3C]
    const __nv_bfloat16* __restrict__ qkv_bias,     // [3C]
    const __nv_bfloat16* __restrict__ rope_cos,     // [T, D]
    const __nv_bfloat16* __restrict__ rope_sin,     // [T, D]
    __nv_bfloat16* __restrict__ q_out,              // [B, H, T, D]
    __nv_bfloat16* __restrict__ k_out,              // [B, H, T, D]
    __nv_bfloat16* __restrict__ v_out,              // [B, H, T, D]
    int B, int T, int C, int H, int D
) {
    // ... CUDA implementation
}
```

### 4.2 TRT plugin wrapper `Code/cpp/src/plugins/fused_attention_pre_plugin.cpp`

```cpp
class FusedAttentionPrePlugin : public nvinfer1::IPluginV2DynamicExt {
    // 实现 enqueue / configurePlugin / getOutputDimensions ... 等接口
    // 在 enqueue 中 launch fused_attn_pre_kernel
};
```

### 4.3 ONNX → TRT 替换路径

ONNX graph rewriter（Python 端）：识别 `(LayerNorm -> Linear -> Reshape -> RoPE_rotate_pair)` 子图，替换为单一 `FusedAttentionPre` 节点。然后 TRT plugin registry 接管。

### 4.4 Feature flag

| 环境变量 | 默认 | 含义 |
|---|---|---|
| `DINOV3_USE_FUSED_ATTN_PRE` | **`0`**（off）| 启用 custom kernel；问题立即 fallback |
| `DINOV3_FUSED_ATTN_PRE_LAYERS` | `all` | `all` / `0-11`（仅前段）/ `0`（仅 1 layer 测试）|

### 4.5 NaN safety

bf16 在 sm_120 上 register pressure 高，可能溢出：
- LayerNorm 中间 mean/var 用 fp32 累加（即使 input bf16）
- attention QK^T 缩放后用 fp32 softmax
- 输出 cast 回 bf16

每层 inference 后插入 `__nanf` 检查（debug build），生产 build 移除。

## 6. 风险

| # | 风险 | Likelihood | Impact | 缓解 |
|---|---|---|---|---|
| 1 | bf16 / sm_120 register pressure 触发 NaN | **High** | High | feature flag default off + 每 commit NaN 检查 |
| 2 | Custom kernel 维护负担超 V1.0.2 周期 | High | Medium | ship 为 research artifact，不进 production default |
| 3 | TRT plugin API 在 10.16/11 间 break | Medium | Medium | 抽象 plugin 接口，升级时统一适配 |
| 4 | profiler 显示 attention 不是 hot | Medium | Low | Gate 0 直接 Defer |
| 5 | cuDNN fused attention 不支持 RoPE → 必须手 roll | Medium | High | 优先方案 C；不支持再 fallback B |
