# ADR-024 · V1.0.4 数据集 1024 → 512 离线 resize

- **Status**: **Implemented**（commit `44c6299`，2026-05-07）
- **Date**: 2026-05-07
- **Related**: V1.0.4 plan §4.1 / §6 acceptance G1

## 1. Context

V1.0.4 新数据集 `Artifacts/datasets/good/` 包含 144 张 **1024×1024** 印花布 JPG 图，命名带 grid 坐标 `r{0-9}c{5-9}`（疑为布料检测多相机/工位 tile 输出）。无 metadata、无 split、无 label。

V1.0.4 推理目标分辨率为 r=**512×512**（patch grid 32×32 = 1024 patches + CLS = 1025 tokens），原因：
- 1024×1024 在 RTX 5080 16 GB VRAM 下推理 OOM（attention QK matrix 4097² × 16 heads × 24 layers ≈ 24 GB，连 b=1 都不可达）
- 512 是 patch-aligned（512/16=32 整除），与 r=518（DINOv3 floor 32×32）token 数一致但内存可控
- 印花布检测生产场景常做 1024 → 512 降采样以提速，与本研究目标一致

**关键决定**（user 2026-05-07 AskUserQuestion）：resize **离线**完成，所有 benchmark 直接读 r=512 数据集，**resize 不算 inference 时序**。

## 2. Decision

实现一次性脚本 `Code/scripts/prepare_r512_dataset.py`，把 144 张 1024×1024 离线 resize 为 512×512 落盘到 `Artifacts/datasets/good_r512/`，并产出 `manifest.json` 含 SHA256 完整性校验。

### 2.1 算法选择

| 维度 | 决策 | 理由 |
|---|---|---|
| Resize 算法 | **PIL `Image.LANCZOS`** | 高质量降采样，4× 下采样保留印花纹理细节避免混叠 |
| JPEG 质量 | **q=95 + optimize** | 保留细节，文件 ~70 KB / 张可接受 |
| 文件命名 | **同源**（保留原始 grid 坐标信息） | 下游 benchmark 可按 grid 复现 |
| 完整性校验 | **每张 SHA256 + 原 src_path 双向追溯** | 数据集可重新部署 / 跨机一致性验证 |
| 错误处理 | open / resize / save **三阶段独立 try-catch** | 单图失败不影响整批 |
| 写入原子性 | manifest 用 tmp + os.replace | 避免 0-byte manifest 文件 |

### 2.2 与项目惯例对齐

- 沿用 V1.0.0 `Artifacts/reports/artifact_manifest_*.json` 的 SHA256 风格
- 路径选 `Artifacts/datasets/`（项目根），不放在 `Code/Artifacts/`（避免与 Code 子目录混）
- ✅ 复用 PIL（项目已通过 `evaluate_engine_pair_on_images.py` 间接依赖）

## 3. 实施

### 3.1 文件

`Code/scripts/prepare_r512_dataset.py` — 296 行 Python，依赖 PIL + hashlib + json（标准库）。

### 3.2 CLI

```bash
python Code/scripts/prepare_r512_dataset.py \
  --input Artifacts/datasets/good \
  --output Artifacts/datasets/good_r512 \
  --target-size 512
```

参数：`--input` / `--output` / `--target-size 512` / `--jpeg-quality 95` / `--manifest` / `--overwrite` / `--ext`。

## 5. Consequences

### Positive

- 144 张 r=512 立即可用 — 后续 ADR-025 ONNX export + ADR-026/027 benchmark 解锁
- Manifest 含 SHA256 — 数据集移植到 Windows / 生产环境时可校验
- 一次性产出，无重复成本

### Negative

- 144 张样本量小（V1.0.0/1/2/3 是 1000 张 ImageNet val），p50 latency 噪声可能略高
  - 缓解：benchmark 配 warmup ≥ 10 + iters ≥ 100 → ≥14400 inferences/run
- 无 split / label，无法用作精度评估目的（只做 latency benchmark）

### Neutral

- 新增脚本 296 行，无 user-facing API 变化，无 breaking change
