# ADR-025 · V1.0.4 r=512 ONNX 导出 + TRT engine 构建

- **Status**: **Code Implemented · Engine Build Awaiting Windows SSH**（commit `591672d` + `4f4ff6c`，2026-05-07）
- **Date**: 2026-05-07
- **Related**: V1.0.4 plan §4.2 / §6 acceptance G2 / §8 Risk #1

## 1. Context

V1.0.0/1/2/3 已在 r=224/336/518 上做过 ONNX 导出 + engine 构建。V1.0.4 新增 r=**512**，需要：
1. 从 PyTorch 模型导出 r=512 的 ONNX（4 输出 binding，opset ≥ 18）
2. 用 trtexec 构建 r=512 的 TRT engine（FP32 + BF16 prefer 必选；FP16 / INT8 SmoothQuant 可选）
3. cos verify > 0.999 vs PyTorch reference 至少 1 张图

**V1.0.4 plan §8 Risk #1（Medium/High）**：`Code/scripts/export_official_dinov3_onnx.py` 当前不支持 `--image-size` 参数（仅 224 hard-coded）。

## 2. Decision

### 2.1 解锁 Risk #1：参数化 `--image-size`

- **修改文件**：`Code/scripts/export_official_dinov3_onnx.py`
- **改动**：加 `--image-size` CLI 参数（默认 224 保持向后兼容），调用 `make_dinov3_vitl16_contract(image_size=N)` 派生新分辨率契约，传递给 `export_model_to_onnx(..., contract=custom_contract)`
- **diff**：12 行新增（`commit 591672d`）
- **验证**：r=512 contract 正确派生 patch grid 32×32 = 1024 patches + CLS = 1025 tokens，output names 不变 (`feat_layer_{4,12,16,20}`)

### 2.2 复用 `build_engine_trtexec.py --image-size`

V1.0.0 的 `Code/scripts/build_engine_trtexec.py` 已支持 `--image-size` 参数（自动派生 `make_dinov3_vitl16_contract` + min/opt/max shape），无需修改。

### 2.3 Engine 命名约定

沿用 V1.0.x 命名规则（resolution 在文件名中）：

| Precision | Engine 文件名 |
|---|---|
| FP32 | `dinov3_vitl16_4out.r512.fp32.engine` |
| BF16 prefer | `dinov3_vitl16_4out.r512.bf16.prefer.engine` |
| FP16（可选） | `dinov3_vitl16_4out.r512.fp16.engine` |
| INT8 SmoothQuant α=0.8（可选） | `dinov3_vitl16_4out.r512.int8.smoothquant_a08.engine` |

### 2.4 Build 参数（沿用 V1.0.1 baseline 配置）

```
--minShapes=pixel_values:1x3x512x512
--optShapes=pixel_values:4x3x512x512
--maxShapes=pixel_values:16x3x512x512
--useSpinWait
```

BF16 加 `--bf16 --precisionConstraints=prefer --layerPrecisions=*:bf16`。

### 2.5 Launcher 脚本

`Code/scripts/build_v104_engines_windows.ps1` 实现自动化：
1. 检查 r=512 ONNX 是否存在，若缺则 `export_official_dinov3_onnx.py --image-size 512`
2. 批量 build FP32 + BF16 prefer engines
3. 可选 `-BuildFp16` 加 FP16 engine
4. 可选 `-SkipFp32 / -SkipBf16` 跳过单 precision

```bash
python Code/scripts/evaluate_engine_pair_on_images.py \
  --reference-engine Artifacts/engines/dinov3_vitl16_4out.r512.fp32.engine \
  --candidate-engine Artifacts/engines/dinov3_vitl16_4out.r512.bf16.prefer.engine \
  --image-root Artifacts/datasets/good_r512 \
  --image-size 512 --max-images 1
```

阈值：所有 4 输出 cos > 0.999（V1.0.4 G2 acceptance）。
