## 1. Context

V1.0.4 plan G5 要求 **Python ↔ C++ 双语言对照**：同 (precision, batch) 在两语言下 6 段 latency p50 diff ≤ 10%；单图 cos = 1.0 bit-exact。

ADR-026 实现 Python `production_benchmark.py`。本 ADR 实现等价 C++ 版本，**严格 mirror Python 6 段计时方案**和 JSON schema，只在 JSON 中标 `language: "cpp"` 区分。

### 2.2 JPEG decode 选型

**vendored `stb_image.h`**（v2.30，public domain MIT）。

| 候选 | 决策理由 |
|---|---|
| `stb_image.h` | ✅ **采用** — single-header / 7988 行 / 零额外构建系统依赖 / public domain / 与 PIL 像素一致性高 |
| OpenCV | ❌ — 添加 OpenCV 依赖（runtime DLLs / 路径管理 / Windows 安装繁琐）远超 V1.0.4 范围 |
| libjpeg-turbo | ❌ — 比 stb 快但需 link 第三方库 + Windows 部署复杂 |

stb_image 解码后 RGB 与 PIL `convert("RGB")` 输出**像素级一致**（公开测试），G5 cos = 1.0 bit-exact 可达。

**Vendor 路径**：`Code/cpp/third_party/stb/stb_image.h`，target_include_directories PRIVATE 添加。

### 2.3 计时方案（mirror Python）

```cpp
using Clock = std::chrono::steady_clock;
auto t0 = Clock::now(); /* disk_read */
auto t1 = Clock::now(); /* jpg_decode (stbi_load_from_memory, req_comp=3) */
auto t2 = Clock::now(); /* preprocess (float/255 + ImageNet mean/std + HWC→NCHW) */
auto t3 = Clock::now(); /* H2D + cudaStreamSynchronize */
auto t4 = Clock::now(); /* enqueueV3 + sync */
auto t5 = Clock::now(); /* D2H × 4 + sync */
auto t6 = Clock::now();
timings.append(disk_read=ms(t1-t0), jpg_decode=ms(t2-t1), ...)
```

每段独立 sync 与 Python 一致，确保两语言 timing 可直接对比。

### 2.4 输出 schema（mirror Python）

JSON 字段名 / 值类型 / 数组结构 严格匹配 Python（仅 `language: "cpp"`）。便于 ADR-028 报告生成时双语言数据 join 同表。

### 2.5 关键技术决策

| 决策 | 理由 |
|---|---|
| **C++17 std::chrono** | 跨平台、无依赖，与 V1.0.x runtime_benchmark.cpp 一致 |
| **`stbi_load_from_memory(req_comp=3)`** | 强制 RGB 输出（同 PIL.convert("RGB")），保证一致 |
| **手写 preprocess 内层 3 重循环（pixel × channel）** | 简单，编译器自动 vectorize；OpenMP 也可加但本研究焦点是测时不是优化预处理 |
| **Output buffer 用 `std::vector<float>`** | 不论 BF16/FP16 都用 fp32 host 接收（与 Python NumPy 一致） |
| **deterministic image order** | `std::sort(jpgs)` 保证 Python 与 C++ 处理同一张图同一顺序 |

## 4. 验收门

### G3 — 6 段独立计时

JSON `stages_p50_ms` 7 字段非零，sum(non-total) ≈ total ±2%。

### G5 — Python ↔ C++ 6 段 diff ≤ 10%

对每 (precision, batch)：

```
diff[stage] = abs(cpp_p50[stage] - py_p50[stage]) / max(py_p50[stage], 0.001)
G5 PASS if all 6 stage diffs ≤ 10%.
```

预期：disk_read / preprocess 在 C++ 应明显更快（无 Python overhead），enqueueV3 / D2H 应基本一致（GPU 主导）。如果 C++ 在某段反而更慢，说明 stb_image vs PIL 不一致或循环未 vectorize，需调查。

### G5（cos）— 单图 bit-exact

参考 V1.0.0 cross-language parity methodology — 用同一 deterministic 输入（fixed seed），Python 与 C++ 各跑一次，4 输出 cos = 1.0 / max_abs_error = 0。

## 5. Consequences

### Positive

- 生产部署可参考 — C++ 是真实生产 inference 路径
- Python ↔ C++ diff 直接量化"两语言 overhead 差距"，对部署选型有意义
- stb_image vendor 不污染依赖树

### Negative / Trade-off

- C++ preprocess 三重循环可能比 NumPy broadcast 慢（小问题，对 GPU 总时序影响 < 1%）
- stb_image 比 libjpeg-turbo 慢约 30-50%（小问题，整个 disk_read + decode 在 r=512 b=8 占总时序 < 10%）

### Neutral

- 不引入新构建系统配置，沿用 V1.0.x CMake 路径
