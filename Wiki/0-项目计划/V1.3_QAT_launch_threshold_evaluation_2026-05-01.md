## 0. ADR-011 § 7 4 条启动门槛原文

V1.3 方案 D（QAT）应在以下条件**全部**满足时启动：

1. **数据集 unblock**：ImageNet val 50K 或同等 distribution 的 ≥ 10K 样本数据集可用。
2. **训练资源到位**：A100/H100 GPU-day 预算 ≥ 5 days，或 RTX 5080 GPU-week ≥ 1 week。
3. **专项时间预算**：1-2 个月专项工程 + 论文写作时间。
4. **下游任务 baseline 就位**：至少一个下游任务（depth / segmentation）的 FP32 baseline + DPT 头训练 pipeline。

## 1. 门槛 1 · 数据集 unblock

### 1.1 当前障碍

`ILSVRC/imagenet-1k` 在 Hugging Face Hub 上是 gated repo，需要在 Meta 网页上申请访问权后才能下载。Windows + RTX 5080 工作站当前 cpolar SSH 隧道下尝试访问得 `403 GatedRepoError`；HF 镜像（hf-mirror.com）也阻塞。

### 1.2 可行性矩阵

| 路径 | 可行性 | 工作量 | 风险 | 推荐度 |
|---|---|---|---|---|
| **A. HF gated 申请** | ⭐⭐⭐ | 1-2 天等待 | 申请被拒（学术目的通常会过） | **优先** |
| **B. Kaggle ImageNet val mirror** | ⭐⭐⭐⭐ | 1 小时下载（50K 张 ~6.4 GB） | Kaggle 镜像可能与官方略有差异 | **优先** |
| **C. Academic Torrents** | ⭐⭐⭐ | 2-4 小时下载 | 速度依赖 seeders；BT 协议在 cpolar 隧道下可能受限 | 备选 |
| **D. 自建下游数据集（NYU Depth V2 / ADE20K）替代** | ⭐⭐ | 3-5 天数据 prep | 与 DINOv3 LVD-1689M 训练 distribution 偏移大 | 仅当 A/B/C 都失败 |
| **E. Imagenette 扩展（10 类 → 100 类）** | ⭐ | 1-2 天 | 仍不是 ImageNet 1000 类完整 distribution | 不推荐 |

### 1.3 推荐先满足路径（成本最低）

**B. Kaggle ImageNet val mirror**：

```bash
pip install kaggle
# Configure Kaggle API token (~/.kaggle/kaggle.json)
kaggle datasets download -d titericz/imagenet1k-validation -p Code/Artifacts/datasets/imagenet_val
unzip imagenet1k-validation.zip
```

成本估算：
- 网络：~6.4 GB 下载（Kaggle CDN 通常 50-100 MB/s，1-2 min）
- 存储：~7 GB（解压）
- 时间：< 1 hour 端到端

**等价性验证**：Kaggle 镜像与官方 ILSVRC2012 val.tar 的 50,000 张 JPEG 应 bit-identical（许多 mirror 是从 Academic Torrents 派生）。SHA256 比对官方 manifest 即可验证。

### 1.4 满足后效果

门槛 1 闭合即可解锁项目所有 § 6 Limitations 中"Imagenette 替代 ImageNet"的退路。除 V1.3 QAT 外，还可重跑现有所有 BF16 / SmoothQuant cosine eval 在完整 ImageNet val 50K 上验证 cos_min 是否仍 ≥ 0.998（项目 §6.1 已 acknowledge 当前 cos_min 是 slight overestimate）。

**门槛 1 难度**：⭐⭐（**最容易先满足**，约 1-2 小时工作）。

## 2. 门槛 2 · 训练资源到位

### 2.1 当前可用硬件

- RTX 5080（Blackwell sm_120, 16 GB VRAM, 300W TDP）— 项目主力机
- macOS（无 GPU 训练能力）
- 学校 / 实验室 A100/H100 — 取决于用户访问权

### 2.2 ViT-L QAT 训练资源估算

QAT fine-tuning 关键参数：
- 模型：DINOv3 ViT-L/16, 304M 参数，FP32 weights ~1.2 GB
- Batch size：QAT 通常 32-64（PTQ initialization 后微调）
- Steps per epoch：ImageNet val 50K / 32 ≈ 1500 steps
- Epochs：1-5（PTQ-init QAT 通常 1-3 epoch 收敛）
- Total steps：1500-7500

| 硬件 | 单步时间估计 | Total time | 成本 |
|---|---|---|---|
| **A100 80GB** | ~0.5s/step (bs=32) | 12-60 min/epoch × 3 = **1-3 GPU-hour** | ~$3-9 (cloud) |
| **H100 80GB** | ~0.3s/step | 0.6-2 GPU-hour | ~$3-12 |
| **RTX 5080 16GB** | ~1.5-2s/step (bs=8 + 4× grad accum) | 4-12 GPU-hour | $0（已有） |
| **Cloud A100 (Lambda Labs)** | 同上 | 1-3 hour | **$1.10/hour × 3 = ~$3** |

### 2.3 推荐先满足路径

**优先级 1：复用项目主力 RTX 5080**

- ✅ 已有硬件，零外部成本
- ⚠️ batch 受 VRAM 约束，需 gradient accumulation 4-8×
- ⚠️ 训练时间长（4-12 GPU-hour）但 acceptable for single experiment

具体可行性：
- ViT-L 304M params × 4 (FP32 weights + grads + optimizer state) = ~5 GB
- Activation peak（bs=8, r224）≈ 4 GB
- Total ≈ 9-10 GB，余 ~6 GB margin in 16 GB → **fits**

**优先级 2：Cloud A100/H100 spot instance**

- 成本：$3-12 一次完整 QAT
- 速度：1-3 hour 端到端
- 推荐 provider：Lambda Labs（学术友好）/ Vast.ai / Coreweave

**优先级 3：学校 GPU 集群**

- 取决于排队 / 调度
- 通常免费但 turnaround 不可控

### 2.4 满足后效果

门槛 2 闭合即可启动单次 V1.3 QAT 实验。如果 PTQ-init QAT 1 epoch 不达 cos ≥ 0.99，可继续到 3-5 epoch；如仍不达，说明 SmoothQuant α=0.8 不是 QAT 的好 init，需重新 calibration。

**门槛 2 难度**：⭐⭐⭐（**第二容易先满足**，零成本路径已就位）。

## 3. 门槛 3 · 专项时间预算

### 3.1 ADR-011 §6.1 估算工作量

- QAT 训练实验：1-3 GPU-hour（门槛 2 已估）
- 训练 hyperparameter sweep（learning rate × loss weight）：3-5 次重跑
- ONNX 重导出 + TRT engine rebuild：每次 30-45 min（已有 pipeline）
- Cosine eval：每个 candidate 15-30 min
- 论文写作（如要 publish）：4-8 周

总工作量：
- 仅 V1.3 实施 + 验证（无论文）：**1-2 周**
- V1.3 实施 + workshop paper（4-6 页）：**1-1.5 月**
- V1.3 实施 + full conference paper：**2-3 月**

### 3.2 推荐先满足路径

**最小可行：1-2 周纯工程实施**

- 复用现有 Paper draft V1.0.0（~7700 词 EN, 100% drafted）作为基础，仅在 §4.4 表 4 增加 V1.3 QAT 行 + §5.3 V1.3 实施记录
- 不需要重写 paper，只需补 1 行 results + 1-2 段 discussion

**完整：1.5 月 V1.3 QAT + paper 完整化**

- 上述 1-2 周工程 + 论文 §2 Literature Review external citation + §4-5 整合 + venue submission

### 3.3 满足后效果

门槛 3 是用户决策项，不是工程项。给出选项后由项目方决定：
- 如果只为 V1.0.1 §12.1 第 3 条达成（cos ≥ 0.99）：**1-2 周**最小可行。
- 如果同步要 venue paper：**1.5-2 月**。

**门槛 3 难度**：⭐⭐⭐⭐（用户决策，非工程不可控）。

## 4. 门槛 4 · 下游任务 baseline 就位

### 4.1 ADR-011 § 7 原文要求

"至少一个下游任务（depth / segmentation）的 FP32 baseline + DPT 头训练 pipeline"。

设计意图：避免"QAT 后 cosine 跨过 0.99 但下游 mIoU/AbsRel 没改善"的 false positive。

### 4.2 可行性矩阵

| 下游任务 | 数据集 | 头训练成本 | DPT 实现可用性 | 推荐度 |
|---|---|---|---|---|
| **NYU Depth V2 (depth estimation)** | 50K 张 RGB-D | DPT-Hybrid 头 ~6 GPU-hour | ✅ DPT 官方 repo + DINOv2 follow-ups | **优先** |
| **ADE20K (semantic segmentation)** | 25K 张 + 150 类 | DPT-Seg 头 ~12 GPU-hour | ✅ MMSeg / Hugging Face | 第二 |
| **KITTI Depth** | 86K 训练 / 7K 验证 | DPT-Hybrid 头 ~10 GPU-hour | 中等 | 备选 |
| **Cityscapes (seg)** | 5K fine-annotated | DPT-Seg 头 ~8 GPU-hour | 中等 | 备选 |

### 4.3 最小可行 baseline pipeline

**NYU Depth V2 + DPT-Hybrid 头**：

```
1. Data prep: NYU Depth V2 official toolkit (~3 hours)
2. DPT head fork: github.com/isl-org/DPT (~1 hour）
3. Replace MiDaS encoder with DINOv3 ViT-L/16 + 4 输出（项目主线产物）
4. Train head with frozen DINOv3 backbone (~6 GPU-hour on RTX 5080)
5. Evaluate AbsRel / RMSE / δ < 1.25 on NYU val
```

工作量：约 **3-5 工作日** 端到端。

### 4.4 与 QAT 的联动

下游 baseline 用法：
- **Pre-QAT**: FP32 backbone + DPT 头 → mIoU/AbsRel baseline
- **PTQ SmoothQuant α=0.8**: INT8 backbone + 同 DPT 头 → 量化后 mIoU 退化
- **Post-QAT**: QAT-trained INT8 backbone + 同 DPT 头 → 验证 cosine 改善是否传导到下游 mIoU

如果 QAT 后 cos_min 跨 0.99 但 NYU AbsRel 仅改善 0.002 (例)，QAT 的工程价值需要重新审视。

### 4.5 推荐先满足路径

**最小可行 (3-5 工作日)**：

1. NYU Depth V2 数据集 prep
2. DPT-Hybrid 头 fork
3. DINOv3 替换 MiDaS encoder
4. 训练 + eval

**门槛 4 难度**：⭐⭐⭐⭐（**最难**，需要新数据集 + 新训练 pipeline + 3-5 工作日）。

## 5. 门槛优先级与启动决策树

### 5.1 难度 / 时间排序

| 排序 | 门槛 | 难度 | 预计时间 |
|---|---|---|---|
| 1（最易） | 1. 数据集 unblock | ⭐⭐ | 1-2 小时 |
| 2 | 2. 训练资源 | ⭐⭐⭐ | 0（已有 RTX 5080）/ $3 cloud |
| 3 | 3. 时间预算 | ⭐⭐⭐⭐ | 1-2 周 / 1.5-2 月（用户决策） |
| 4（最难） | 4. 下游 baseline | ⭐⭐⭐⭐ | 3-5 工作日 |

### 5.2 启动决策树

```
是否要启动 V1.3 QAT?
├── 不启动（推荐 default）
│    └── R2 应急方案（cos_mean 视角）作为 V1.0.1 主交付
│         详见 R2_emergency_acceptance_analysis_V1.0.0.md
│
└── 启动
     ├── 仅 V1.3 实施（不发论文）
     │   └── 预算：1-2 周 + Kaggle ImageNet (1 hour) + RTX 5080 (12 hour) + 跳过下游 baseline（接受 cosine 单 metric）
     │       总成本：~$0 + 2 工作周
     │
     ├── V1.3 + workshop paper（4-6 页）
     │   └── 预算：1.5 月 + Kaggle ImageNet + RTX 5080 + NYU Depth baseline + paper 增补
     │       总成本：~$0 + 6 工作周
     │
     └── V1.3 + full conference paper（10-12 页）
         └── 预算：2-3 月 + Kaggle ImageNet + Cloud A100（更快）+ NYU + ADE20K baselines + paper 完整化
             总成本：~$50-200 cloud + 10-12 工作周
```

### 5.3 推荐处置（本文作者建议）

**短期（1 周内）**：

1. ✅ 接受 R2 应急方案（cos_mean ≥ 0.97 视角）作为 V1.0.1 主交付路径。
2. ⏳ 推迟 V1.3 启动，等用户决策。

**中期（如用户决定启动）**：

按"先易后难"顺序满足门槛：
1. 周一：Kaggle ImageNet val 下载 (1 hour) → 门槛 1 闭合
2. 周一-周三：RTX 5080 上跑 V1.3 QAT 第一次实验 (12-24 hour) → 验证 cos_min 是否跨 0.99
3. **决策点**：如 cos_min < 0.99，QAT init 不足 → 调 hyperparameter / 换 calibration / 加更多 epoch
4. 周四-周五：如 cos_min ≥ 0.99，启动 NYU Depth baseline（3-5 工作日）
5. 第 2-3 周：完成 baseline + QAT 下游验证 + paper §4-5 增补

**长期（如要发论文）**：

按 §5.2 决策树选 workshop / full conference 路径，分别 6 / 10-12 工作周。

## 6. 与 V1.0.1 §12.1 验收清单的关系

V1.3 QAT 是 V1.0.1 §12.1 第 3 条 "INT8 cos ≥ 0.99" 的**严格阈值穿透路径**。本期 V1.0.1 已通过 R2 应急方案（cos ≥ 0.97 视角）给出 acceptable 交付路径；V1.3 启动是**"严格 0.99 + V1.3 stretch goal" double-达成** 的可选 future work。

**§12.1 不强制 V1.3 启动**：R2 应急方案在 cos_mean 视角已 4/4 ≥ 0.97 达成，足够构成 acceptable 交付。
