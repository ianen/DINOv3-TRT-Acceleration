## 1. V1.0.1 frozen baseline（出发点）

| 维度 | V1.0.1 实测（real ImageNet val 1000） | 来源 |
|---|---|---|
| BF16 prefer trtexec GPU compute peak | **3.86×** / r518 b8 | `formal_benchmark_matrix.csv` 第 67 轮 |
| BF16 prefer cpp end-to-end peak | **3.40×** / r518 b8 | 同上 |
| BF16 prefer cos_min | **0.9977** → R1_PASS_strict | `imagenet50k_post_download_summary.json` |
| INT8 SmoothQuant α=0.8 cos_min | **0.9727** → R2_PASS_emergency（R1 未达，缺口 0.022）| 同上 |
| 跨语言一致性 | 224/336/518 × FP32/BF16 batch 1 全部 **bit-identical** | V1.0.1 G3 |
| memory-bound 边界 | r336 b16+ / r518 b16 跌到 ~1.0×（FP32 与 BF16 都 BW saturated）| ADR-010 备注 |

**关键观察**：V1.0.1 是**单 stream 顺序推理**架构 — 单 cudaStream，无 pinned memory，无 CUDA Graph，单 ExecutionContext，无多 optimization profile，无 sparsity，无 custom kernel。所有"算法层精度优化"已穷尽（V1.0/V1.1/V1.2 mixed-precision 全部 negative，仅 BF16 prefer R1 达标），但**runtime mechanics 维度尚未触及**。

## 2. V1.0.2 SMART 目标

| Goal | Metric | V1.0.1 baseline | **V1.0.2 target** | Stretch | 可行性论证 |
|---|---|---|---|---|---|
| **G1 低 batch 延迟** | cpp end-to-end p50 (r518 b1) | 1.0× ref | **≥ 4.5×** | 5.2× | CUDA Graphs 消除 50–200 μs launch overhead；b1 时 launch 占 ~10–15% |
| **G2 高 batch 吞吐** | cpp end-to-end p50 (r224 b32) | 3.20× | **≥ 4.0×** | 4.5× | Pinned + async overlap (~1.15×) + 2:4 sparsity (~1.3×) 复合 |
| **G3 攻克 memory-bound 墙** | cpp end-to-end p50 (r518 b16) | ~1.0× (BW saturated) | **≥ 1.6×** | 2.0× | 2:4 sparsity 半 weight bandwidth — 直接破 BW wall |
| **G4 多 stream 聚合吞吐** | aggregate img/sec at N=4 streams | N/A | **≥ 3.0× single-stream** | 3.5× | RTX 5080 84 SM，单流占 50–70%；多流填空闲 SM |
| **G5 BF16 严格精度保留** | cos_min on real ImageNet val 1000 | 0.9977 | **≥ 0.997** | ≥ 0.999 | Graphs/streams/pinned/profiles bit-exact；只有 sparsity & FP8 风险（独立 gate）|
| **G6 build 确定性** | engine SHA256 跨 3 次重建可复现 | N/A | **100% with persistent timing cache** | — | 移除 "first build slow"；论文复现支持 |

### 旗舰目标（headline）

**cpp end-to-end ≥ 5.0× speedup at r518 b8（BF16 prefer + CUDA Graphs + pinned memory + 2:4 sparsity），cos_min ≥ 0.99 on real ImageNet val 1000**。

复合估算：3.40 × 1.10（Graphs+pinned bit-exact）× 1.30（2:4 sparsity weight-bound）≈ **4.86×**；多 profile tactic 优化 → **5.0×** 中位数可达。

## 3. 8 条 untapped 优化向量（从 codebase audit 得出）

| # | 向量 | 类别 | 预期增益（叠乘）| 精度风险 |
|---|---|---|---|---|
| 1 | CUDA Graphs（capture + replay） | runtime | 1.05–1.15× at b≤4 | bit-exact，无风险 |
| 2 | Pinned host memory + async H2D/D2H overlap | runtime | 1.05–1.15× across configs | bit-exact，无风险 |
| 3 | Persistent timing cache + 多 optimization profiles | build | 1.05–1.10× extreme batch | bit-exact，无风险 |
| 4 | Multi-stream concurrent inference (StreamPool) | runtime | aggregate 2.5–3.5× at N=4 | bit-exact，无风险 |
| 5 | 2:4 structured sparsity (NVIDIA ASP) on attention QKV + MLP | weights | 1.2–1.5× weight-bound 层 | cos_min 可能掉 0.005–0.020 |
| 6 | FP8 refined per-tensor + per-channel scaling | quantization | 1.2–1.6× on Hopper-style FP8 | cos_min 风险中等 |
| 7 | Custom CUDA fused kernel（LayerNorm+RoPE+QKV proj） | kernel | 1.10–1.30× attention-bound 层 | NaN 风险高 |
| 8 | TRT 10.16.1 升级（Week 0 前置 milestone）| infra | sparsity 成熟度依赖 | 可能 break BF16 numbers |

## 4. 排序与节奏（8–12 周）

| Week | 内容 | ADR | Risk | Cos gate |
|---|---|---|---|---|
| **0** | TRT 10.13.2.6 → 10.16.1 升级 + V1.0.1 数字复现验证（前置 milestone）| ADR-014 | 中 | ≥ 0.997 |
| **1–2** | Quick wins：CUDA Graphs（item 1）+ Pinned Memory（item 2）+ Persistent Timing Cache + Multi-Profile（item 3）| ADR-012, ADR-013 | 低 | bit-exact，无 gate |
| **3–4** | Multi-stream concurrent pool（item 4）| ADR-015 | 中（contention）| bit-exact |
| **5–6** | 2:4 structured sparsity（item 5）— **首个有 cos 风险的 V1.0.2 项** | ADR-016 | 中（cos 风险）| ≥ 0.99 严格 gate |
| **7–8** | FP8 refined scaling（item 6）— opportunistic | ADR-017 | 中 | ≥ 0.99 |
| **9–12** | Custom CUDA kernel（item 7）— **only if** profiler post-week-8 显示 attention 仍 ≥ 20% unfused | ADR-018 | 高 | feature flag default off + ≥ 0.99 gate |

**为什么先 quick wins 而非 sparsity**：CUDA Graphs / pinned memory 是 bit-exact 改动，可作为后续 sparsity / FP8 / 自定义 kernel 的稳定基线。如果先做 sparsity 再做 graphs，graphs 引入的 launch overhead 节省可能被 sparsity 引入的精度噪声掩盖。

## 8. Risk Register（Top 5）

| # | Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|---|
| 1 | CUDA Graphs 不兼容动态 shape | Medium | High | 每 (batch, resolution) 独立 capture 一个 Graph；缓存 6 个；fallback 非 Graph 路径 |
| 2 | 2:4 sparsity 把 cos_min 拉到 < 0.99 | Medium | High | 先 per-layer ablation；只用耐受层；不可恢复则降为 stretch result |
| 3 | Multi-stream contention（L2/HBM 共享 sub-linear）| High | Medium | benchmark N∈{1,2,4,8}；< 2.5× 就 ship 2-stream，记 ADR |
| 4 | Custom kernel 在 bf16 / sm_120 register pressure 下 NaN | Medium-High | High | feature flag default off + per-commit cos gate；不稳定 → 留 research artifact |
| 5 | TRT 10.16.1 升级 break BF16 numbers | Medium | Medium | TRT 升级作为 V1.0.2-pre 独立 milestone；复现 V1.0.1 数字否则 stay on 10.13.2.6 |
