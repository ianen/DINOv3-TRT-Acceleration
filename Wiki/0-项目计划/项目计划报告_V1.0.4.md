- **状态**：Proposed（2026-05-07 起草）

## 1. V1.0.4 概述与定位

### 1.1 背景

V1.0.0 / V1.0.1 / V1.0.2 / V1.0.3 系列的研究焦点是**精度与吞吐的边界刻画**：
- V1.0.0/1：12 候选精度 sweep，确定 BF16 prefer 主交付（trtexec 3.86× / cpp 3.40× vs FP32）
- V1.0.2：5-vector PTQ precision wall 实证，INT8/FP8/2:4-sparse 全墙
- V1.0.3：throughput-oriented serving + GPU saturation 实证（720 qps / SM 98.74%）

V1.0.4 切换到**生产环境实测**视角：在一个真实业务场景的数据集（印花布检测）上跑 r=512 inference，并测量**端到端时序分解**（不仅 GPU compute，还含 disk I/O + JPEG decode + preprocess），输出可用于生产部署评估的 benchmark 报告。

### 1.2 与 V1.0.3 的区别

| 维度 | V1.0.3 | V1.0.4 |
|---|---|---|
| 焦点 | Throughput-oriented serving + saturation | Production-environment inference + 时序分解 |
| 分辨率 | 224 / 336 / 518（已有 engines） | **r=512 新增（patch grid 32×32 = 1025 tokens）** |
| 数据 | 合成 sin input + ImageNet val 1000 | **印花布真实数据 144 张** |
| 测量口径 | aggregate qps + SM utilization | **端到端 6 段独立 latency**（disk + decode + preprocess + H2D + inference + D2H） |
| 候选精度 | BF16 prefer 主线 + INT8 SmoothQuant | FP32 + BF16 必选；FP16 / INT8 可选扩展 |
| 实施层 | 多 context pool / Triton（throughput） | **Python + C++ 双语言 production_benchmark**（latency + 时序分解） |

### 1.3 用户关键决定（AskUserQuestion 2026-05-07 确认）

| 决定 | 说明 |
|---|---|
| **resize 离线完成** | 144 张 1024×1024 离线 resize 为 512×512，落盘形成新数据集 `Artifacts/datasets/good_r512/`；resize 不算 inference 时序，所有 benchmark 直接读 r=512 数据集 |
| **production 含预处理** | 6 段独立计时：disk read / JPEG decode / preprocess (norm + NCHW) / H2D / enqueueV3 / D2H |
| **必选精度 FP32 + BF16** | FP16（V1.0.0 NaN 负例可在 r=512 重验证）/ INT8 SmoothQuant α=0.8（V1.0.2 R2 候选）作可选扩展 |
| **batch {1, 4, 8, 16}** | 4 个 batch 点；不测 b=32（r=512 b=32 在 16GB VRAM 风险高） |

## 3. 数据集

### 3.1 来源

`Artifacts/datasets/good/` — 144 张 1024×1024 JPG 印花布图像。

**文件命名约定**：`{batch}_{timestamp}_sp_{frame:06d}_{value}_r{row}c{col}.jpg`
- 例：`1_20260416140116_sp_000001_0.06_r0c6.jpg`
- batch ID = 1（数据采集批次）
- timestamp = 20260416140116
- frame index = sp_000001 ~ sp_000108
- metric value = 0.06 ~ 8.31（疑为缺陷度量）
- grid coord = r{0-9}c{5-9}（疑为多相机/工位 grid 输出）

### 3.2 离线 resize 协议

新建数据集 `Artifacts/datasets/good_r512/`，144 张同名 JPG，仅尺寸 1024 → 512。

**resize 算法**：PIL.Image.LANCZOS（高质量降采样，避免印花纹理混叠）
**质量**：JPEG quality=95（保留印花细节）
**完整性校验**：每张图 SHA256 写入 `Artifacts/datasets/good_r512/manifest.json`

### 3.3 数据使用约定

- benchmark warmup 阶段：固定从 144 张中任选 1 张反复跑 ≥ 10 次
- benchmark timed 阶段：循环遍历 144 张多遍至累计 ≥ iters 次
- 不分 train/val（无 label，纯 inference benchmark）

### 4.1 ADR-024 — 数据集离线 resize

**输入**：`Artifacts/datasets/good/*.jpg` (144 张 1024×1024)
**输出**：
- `Artifacts/datasets/good_r512/*.jpg` (144 张 512×512)
- `Artifacts/datasets/good_r512/manifest.json` (含 SHA256 + 原始图片路径映射)

**新建文件**：`Code/scripts/prepare_r512_dataset.py` (~80 行)
**核心逻辑**：
```python
for src_jpg in sorted(input_dir.glob("*.jpg")):
    img = Image.open(src_jpg).convert("RGB")
    if img.size != (1024, 1024):
        log_warn(f"size mismatch: {src_jpg.name} = {img.size}")
    img_512 = img.resize((512, 512), Image.LANCZOS)
    dst = output_dir / src_jpg.name
    img_512.save(dst, "JPEG", quality=95)
    sha256 = hashlib.sha256(dst.read_bytes()).hexdigest()
    manifest.append({"file": src_jpg.name, "sha256": sha256, ...})
```

**验收**：
- `ls Artifacts/datasets/good_r512/*.jpg | wc -l` = 144
- 任取 1 张 PIL.Image.size = (512, 512)
- `manifest.json` 含 144 条 SHA256 entry

### 4.3 ADR-026 — Python production_benchmark 含 6 段独立计时

**新建文件**：`Code/scripts/production_benchmark.py` (~250 行)

**CLI**：
```
python production_benchmark.py \
  --engine Artifacts/engines/dinov3_vitl16_4out.r512.bf16.prefer.engine \
  --dataset Artifacts/datasets/good_r512 \
  --batch-size 8 \
  --warmup 10 \
  --iters 100 \
  --output Artifacts/reports/v104_runs/r512_bf16_b8_py.json
```

**核心计时逻辑**（每张图独立计时 6 段）：
```python
for i in range(iters):
    img_path = images[i % len(images)]

    t0 = time.perf_counter()
    raw = img_path.read_bytes()                       # disk_read
    t1 = time.perf_counter()

    img = Image.open(io.BytesIO(raw)).convert("RGB")  # jpg_decode
    t2 = time.perf_counter()

    arr = np.asarray(img, dtype=np.float32) / 255.0   # preprocess
    arr = (arr - mean) / std
    arr = arr.transpose(2, 0, 1)[None]                # NCHW
    t3 = time.perf_counter()

    cudart.cudaMemcpyAsync(d_in, arr.ctypes.data, ..., stream)  # H2D
    t4 = time.perf_counter()

    context.execute_v2(bindings)                       # enqueueV3
    cudart.cudaStreamSynchronize(stream)
    t5 = time.perf_counter()

    cudart.cudaMemcpyAsync(h_out, d_out, ..., stream) # D2H
    cudart.cudaStreamSynchronize(stream)
    t6 = time.perf_counter()

    record_latencies(t0..t6)  # 6 段独立 ms
```

**输出 JSON schema**：
```json
{
  "engine": "...", "batch_size": 8, "image_size": 512,
  "warmup": 10, "iters": 100,
  "stages_p50_ms": {
    "disk_read": 0.50, "jpg_decode": 4.80,
    "preprocess": 1.20, "h2d": 0.80,
    "enqueueV3": 32.00, "d2h": 0.40,
    "total": 39.70
  },
  "stages_p95_ms": { ... },
  "throughput_imgs_per_sec": 25.2,
  "per_image": [ /* full 100-iter trace */ ]
}
```

**复用基础设施**：V1.0.0 `Code/scripts/evaluate_engine_pair_on_images.py` 的图像加载 + preprocess 逻辑可直接抽用。

### 4.4 ADR-027 — C++ production_benchmark 含 6 段独立计时
