## V1.0.3 Honest Assessment

V1.0.3 was framed as "throughput-oriented serving + full GPU utilization." The **utilization half is empirically complete and exceeds plan thresholds**. The **throughput half is blocked by external user-side actions** that the plan §8 explicitly anticipated as Risk #1 and Risk #2.

This is the *plan working* — the risks were predicted, materialized, and have documented unblock paths. It is not a project failure; it is a clean handoff awaiting one of two user decisions (install Docker OR upgrade TRT) before the throughput numbers can move.

In the meantime, the G7 saturation data is **paper-ready** and provides the strongest empirical case yet for V1.3 QAT — the BF16 dense path is provably saturated; further speedup requires Tensor Core ceiling break via INT8/FP8/2:4 sparse, which the V1.0.2 PTQ frozen-negative results already proved needs QAT.
