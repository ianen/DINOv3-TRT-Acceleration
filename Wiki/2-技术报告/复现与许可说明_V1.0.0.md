日期：2026-04-30

## 范围

本说明收束 V1.0.1 项目计划中 G5 / M7 的可复现性交付：代码入口、数据口径、正式 artifact manifest、DINOv3 license 副本与 "Built with DINOv3" 标注。

当前可完整复现的正式口径是 RTX 5080 / TensorRT 10.13.2.6 / 224x224 / Imagenette2-320 val。完整 ImageNet val 仍受 Hugging Face gated download `403 GatedRepoError` 阻塞；授权可用后，应复用同一套 manifest/eval/calib 入口替换数据源。

## 许可与标注

本项目使用 Meta DINOv3 ViT-L/16 LVD-1689M，模型与相关材料受 DINOv3 License 约束，而不是 Apache/MIT 这类宽松开源协议。

已落地的许可交付：

- 仓库级 license 副本：`LICENSES/DINOv3_LICENSE.md`
- 本地 artifact 来源副本：`Code/Artifacts/source/dinov3/LICENSE.md`
- HF 权重快照来源副本：`Code/Artifacts/weights/dinov3-vitl16-pretrain-lvd1689m/LICENSE.md`

报告和 README 中保留标注：Built with DINOv3。对外发布论文、报告或派生成果时，应继续保留对 DINOv3 Materials 的使用 acknowledgement，并遵守 DINOv3 License 中关于再分发、贸易管制、禁止反向工程与终止条款的要求。

不得提交到 git 的内容：

- Hugging Face token 或其他访问凭证
- `model.safetensors`、`.onnx`、`.engine`、`.timing.cache`
- ImageNet、Imagenette 或其他数据集原始文件
- `Code/Artifacts/` 下的大体积生成物

## 轻量复现

本地机器不要求 CUDA/TensorRT，可先验证代码、报告生成器和同步逻辑：

```bash
cd Code
.venv/bin/python -m pytest tests/test_benchmark_figures.py tests/test_benchmark_matrix.py tests/test_remote_sync.py
.venv/bin/python -m ruff check src/dinov3_trt/reports scripts/build_benchmark_figures.py scripts/build_benchmark_matrix.py tests/test_benchmark_figures.py tests/test_benchmark_matrix.py
.venv/bin/python -m mypy src/dinov3_trt/reports scripts/build_benchmark_figures.py scripts/build_benchmark_matrix.py tests/test_benchmark_figures.py tests/test_benchmark_matrix.py
```

同步代码和文档到 Windows RTX 5080（local → remote）：

```bash
cd Code
.venv/bin/python scripts/sync_remote_windows_repo.py --host windows-pc --no-git-init
```

把远端生成的 `Artifacts/reports/`（text-only：`.json/.md/.csv/.svg/.png/.jpg/.log/.txt`）回拉到本地（remote → local）：

```bash
cd Code
.venv/bin/python scripts/sync_remote_windows_repo.py --pull-reports --host windows-pc
```

`--pull-reports` 会在远端用 PowerShell `System.IO.Compression` 打 zip（按扩展名白名单过滤，不带 engine/onnx/weights 等大体积二进制），scp 拉到本地临时目录，原子解压到 `Code/Artifacts/reports/`，并清理远端 zip。zip-slip / 绝对路径条目会被解压逻辑拒绝。这条命令安全幂等，反复跑会刷新本地 reports 与最新远端一致。

## RTX 5080 正式复现入口

在 Windows 侧 `D:\WorkPlace\ZMP\DINOv3-TRT-Acceleration\Code` 下执行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\run_formal_hf_pipeline_windows.ps1
```

```powershell
.venv\Scripts\python.exe scripts\build_int8_tradeoff_figure.py --reports-dir Artifacts\reports --output-dir Artifacts\reports\figures
```

`build_cosine_summary_figure.py` 直接读取多分辨率 `eval_imagenette1000_*_fp32_vs_bf16_prefer.json`，生成 `benchmark_bf16_cosine_min.svg` 和 `benchmark_bf16_cosine_mean.svg`：4 个 output binding × 224/336/518 三档分辨率的 grouped 柱状图，y 轴自动缩放到 cosine 高密集区间（典型 0.997-1.0）。如某个分辨率的 eval 报告暂未生成，可加 `--allow-missing` 跳过。

`build_int8_tradeoff_figure.py` 读取 BF16 prefer + 三种 INT8 partial 的 eval JSON 与 locked trtexec speedup JSON，输出 `benchmark_bf16_vs_int8_tradeoff.svg`：cosine × latency speedup 的散点图（默认取 batch 8 的 `feat_layer_20` cosine_mean），并标注 G2 阈值线（cos = 0.99 垂直 + speedup = 2.2× 水平）与右上 ideal region 阴影，把"量化范围 vs 精度 vs 速度"的 trade-off curve 一图说清。

## 多分辨率（336 / 518）补点

主路径仍是 224x224；336 与 518 是 G4 多分辨率 benchmark 的扩展。每个分辨率独立 ONNX 与 engine（197 / 442 / 1025 token 数依赖固定输入），timing cache 也独立。

### ONNX 导出 + 检查

```powershell
python scripts\export_hf_dinov3_onnx.py --model-path Artifacts\weights\dinov3-vitl16-pretrain-lvd1689m --local-files-only --output Artifacts\onnx\dinov3_vitl16_4out.r336.onnx --image-size 336 --validate-no-if
python scripts\inspect_onnx.py Artifacts\onnx\dinov3_vitl16_4out.r336.onnx

python scripts\export_hf_dinov3_onnx.py --model-path Artifacts\weights\dinov3-vitl16-pretrain-lvd1689m --local-files-only --output Artifacts\onnx\dinov3_vitl16_4out.r518.onnx --image-size 518 --validate-no-if
python scripts\inspect_onnx.py Artifacts\onnx\dinov3_vitl16_4out.r518.onnx
```

### TensorRT engine 构建

r336（profile 默认 `min=1, opt=8, max=32`，与 224 一致）：

```powershell
python scripts\build_engine_trtexec.py --onnx Artifacts\onnx\dinov3_vitl16_4out.r336.onnx --engine Artifacts\engines\dinov3_vitl16_4out.r336.fp32.engine --precision fp32 --image-size 336 --workspace-gb 4 --timing-cache Artifacts\engines\dinov3_vitl16_4out.r336.fp32.timing.cache --run-inference
python scripts\build_engine_trtexec.py --onnx Artifacts\onnx\dinov3_vitl16_4out.r336.onnx --engine Artifacts\engines\dinov3_vitl16_4out.r336.bf16.prefer.engine --precision bf16 --image-size 336 --workspace-gb 4 --timing-cache Artifacts\engines\dinov3_vitl16_4out.r336.bf16.prefer.timing.cache --precision-constraints prefer --layer-precision *:bf16 --run-inference
```

r518 用**保守 profile** `min=1, opt=2, max=4`（受 16 GB VRAM 限制，1025 token 下 attention `[B,H,1025,1025]` 占用更高）：

```powershell
python scripts\build_engine_trtexec.py --onnx Artifacts\onnx\dinov3_vitl16_4out.r518.onnx --engine Artifacts\engines\dinov3_vitl16_4out.r518.fp32.engine --precision fp32 --image-size 518 --min-batch 1 --opt-batch 2 --max-batch 4 --workspace-gb 4 --timing-cache Artifacts\engines\dinov3_vitl16_4out.r518.fp32.timing.cache --run-inference
python scripts\build_engine_trtexec.py --onnx Artifacts\onnx\dinov3_vitl16_4out.r518.onnx --engine Artifacts\engines\dinov3_vitl16_4out.r518.bf16.prefer.engine --precision bf16 --image-size 518 --min-batch 1 --opt-batch 2 --max-batch 4 --workspace-gb 4 --timing-cache Artifacts\engines\dinov3_vitl16_4out.r518.bf16.prefer.timing.cache --precision-constraints prefer --layer-precision *:bf16 --run-inference
```

r518 batch 8 用**独立的 b8 engine**（`min=1, opt=4, max=8`，timing cache 单独，避免与 max=4 engine 冲突）：


## C++ runtime（多分辨率）

C++ runtime tools 都接受可选 `--image-size N` flag（`--image-size=N` 等价），不传时回到 224 默认；输入 shape 校验从 hard-coded `[B,3,224,224]` 放宽为 `H=W、C=3、H≥patch_size`，DINOv3 floor(H/patch_size) 语义适配 518=32×16+6。

构建（首次或代码变更后必跑）：

```powershell
powershell -ExecutionPolicy Bypass -File scripts\build_cpp_trt_inspector_windows.ps1
```

smoke / benchmark / parity（举例 r336 BF16 prefer）：

```powershell
build\cpp-trt-inspect-msvc\dinov3_trt_runtime_smoke.exe Artifacts\engines\dinov3_vitl16_4out.r336.bf16.prefer.engine 1 --image-size 336
build\cpp-trt-inspect-msvc\dinov3_trt_runtime_benchmark.exe Artifacts\engines\dinov3_vitl16_4out.r336.bf16.prefer.engine 1 10 50 --image-size 336
.venv\Scripts\python.exe scripts\compare_cpp_python_parity.py --engine Artifacts\engines\dinov3_vitl16_4out.r336.bf16.prefer.engine --dump-dir Artifacts\reports\cpp_python_parity_dump_r336_bf16_prefer --output Artifacts\reports\cpp_python_parity_r336_bf16_prefer_b1.json --batch-size 1 --image-size 336
```

r518 同理把 engine 路径换成 `r518.bf16.prefer.engine`（profile max=4，可用 batch 1/2/4）或 `r518.bf16.prefer.b8.engine`（profile max=8，可用 batch 1-8），并把 `--image-size` 改为 `518`。

## 数据替换流程

当前真实图片 eval/calib 使用公开 Imagenette2-320 val：

- eval：`Artifacts/manifests/imagenette_selected_eval_1000.json`
- calib：`Artifacts/manifests/imagenette_selected_calib_500.json`

完整 ImageNet val 授权可用后，先把 HF parquet validation shards 放到 `Artifacts/datasets/imagenet-1k-hf/data/`，再转换为 ImageNet-style class folder：

```bash
cd Code
python -m pip install -e ".[data]"
python scripts/export_hf_imagenet_parquet_images.py \
  --parquet-dir Artifacts/datasets/imagenet-1k-hf/data \
  --glob "validation-*.parquet" \
  --output-root Artifacts/datasets/imagenet-val \
  --manifest-output Artifacts/manifests/imagenet_val_all.json \
  --split validation
```

生成互斥 eval/calib manifest：

```bash
python scripts/prepare_image_subset_manifests.py \
  --image-root Artifacts/datasets/imagenet-val \
  --eval-output Artifacts/manifests/imagenet_eval_1000.json \
  --calib-output Artifacts/manifests/imagenet_calib_500.json \
  --eval-count 1000 \
  --calib-count 500
```

替换 manifest 后重跑 BF16 eval、ModelOpt calibration 和正式 summary/matrix，即可把报告口径从 Imagenette 更新为完整 ImageNet val。
